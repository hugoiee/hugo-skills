# 把一句音乐创意变成可直接出歌的 Suno 成品

## 这个 skill 解决什么问题

Suno 吃的不是"一句模糊的愿望"——它吃的是**精准的风格描述 + 各归其位的歌词结构**。把"帮我写首伤感的歌"直接丢给 Suno，常见三种崩法：

- **提示词太泛**：「一首好听的伤感情歌」→ 出来千篇一律、塑料感十足的"AI 味"，没有记忆点。
- **风格堆太多 / 多流派打架**：「流行 + 摇滚 + 电子 + 古风 + 爵士」全塞进去 → 混浊、四不像，AI 不知道听谁的。
- **没结构标签 / Style 和歌词混填**：歌词里夹风格描述、风格框里写故事 → AI 分不清哪句要唱、歌曲结构松散垮掉。

这个 skill 把"中文创意 → Suno 成品"压成**一条五步管道**：

```
Step 1  厘清创作意图          拆已知要素，关键信息缺失先问 1-2 个澄清问题
Step 2  写 Style 提示词        英文、≤240 字符，锚定一个主流派 + 情绪/乐器/人声/节奏/制作
Step 3  写结构化歌词          带 [Verse]/[Chorus] 标签，语言随需求，≤3000 字符（纯音乐则跳过）
Step 4  取标题 + 选模式/版本   Custom vs Simple，提示当前主力版本与字符限制
Step 5  调参迭代             两版选一版，再用 Extend/Replace/Cover/Persona 局部修
```

> 你的角色是**制作人 + 词曲搭档**，不是点歌机。目标不是"把用户的话复述成英文"，而是"让 Suno 唱出用户脑子里那首歌"。

## 核心心法（贯穿全流程）

1. **Style 用英文，歌词随用户语言** —— Suno 对英文风格描述理解最稳、出活最准；但歌词要中文歌就写中文词，要日文就写日文，别擅自改语言。
2. **一首歌只锚定一个主流派** —— 选一个 anchor genre 定盘，其余风味用乐器、情绪、制作质感去调，别把五个流派平铺堆在一起。
3. **画一个"声音场景"，而不是堆关键词** —— v4.5+ 能读懂连贯描述：开头什么质感、怎么递进、副歌怎么炸、结尾怎么收。一句有画面的描述胜过十个孤立标签。
4. **Style 和歌词各归其位** —— 风格、情绪、乐器、人声、BPM 进 Style 框；所有要被唱出来的字进 Lyrics 框。两个框别串味。
5. **结构标签是骨架** —— `[Verse] [Chorus] [Bridge]` 让歌有章法；标签独占一行，AI 才认得出段落转换。
6. **一行一个意思、控制音节** —— 一行别塞太多字，否则人声唱得赶、咬字糊；副歌短而重复才抓耳。
7. **第一遍不完美是常态** —— 先生成两版选更稳的那版，再用 Extend / Replace Section / Cover 局部修，别每次推倒重来烧 credit。

## Step 1 — 厘清创作意图 & 按需反问

**先别急着写。** 第一步是读懂用户到底想要一首什么歌，并判断信息够不够。

### 拆出已知要素

把用户的创意拆进下面的格子，看哪些已知、哪些为空：

```
流派    什么曲风？        （流行 / 民谣 / 电子 / 嘻哈 / 摇滚 / 国风 / lo-fi…）
情绪    什么气质/场景？    （治愈 / 致郁 / 燃 / 深夜 / 通勤 / 婚礼…）
人声    有没有 & 什么人声？（男声 / 女声 / 童声 / 合唱 / 纯音乐无人声）
语言    歌词用什么语言？   （中文 / 英文 / 日文 / 双语…）
节奏    快慢 / BPM？      （抒情慢歌 / 律动中速 / 蹦迪快歌）
参考    "像谁的感觉"？     （描述化，不直接点名歌手——见 references/style-and-vocab.md）
用途    长度/场景          （完整歌 / 短视频 BGM / 副歌 demo）
```

### 关键信息缺失 → 先反问（精简，给选项）

主体情绪/流派/人声/语言这几样里若有关键项空着，**先用 1-2 个澄清问题问清楚再写**，别自己脑补：

```
你这首「分手后的歌」想要哪种走向？
A. 抱头痛哭的抒情慢歌（钢琴/弦乐）
B. 释怀向上的中速流行
C. 带点恨意的暗黑电子/嘻哈
人声要男声还是女声？歌词用中文还是英文？
```

> **例外**：用户已经说得很具体（流派、情绪、人声、语言都清楚），或明确说"你看着办/随便来一版"，就跳过反问直接进 Step 2，并在输出的「说明」里讲清哪些是我替他定的。

### 纯音乐分支

如果用户要的是 BGM / 纯音乐 / 无人声：**确认 instrumental**，跳过 Step 3 的歌词（或只用 `[Instrumental]` 段落描述），并在 Style 里写明 `instrumental`。

---

## Step 2 — 写 Style 提示词（英文，≤240 字符）

这是决定"出片率"的一步。Style 框是 Suno 的总指挥，**写英文、锚定一个主流派、画出声音场景**。

### 配方

```
[anchor genre] + [mood 1-2 词] + [关键乐器] + [人声特征] + [节奏/BPM(可加 key)] + [制作质感]
```

- **中文创意 → 英文专业术语**：流派、情绪、乐器、唱法、制作效果都查 `references/style-and-vocab.md` 的中→英词库，别用机翻硬译。
- **一个主流派定盘**：`dark R&B` 就别再加 `+ rock + EDM + folk`；想要别的风味，用乐器和制作质感去点缀。
- **情绪 1-2 个词**：别堆 `sad, melancholic, sorrowful, gloomy` 一串同义词，稀释指令。
- **可写"走向"**：v4.5+ 读得懂 `starts sparse with soft piano, builds to an anthemic chorus, fades out gently`。
- **不想要的元素**：不写进描述句，用 Exclude styles（负面提示，见 `references/suno-features.md`）。
- **数字符 ≤240**：写完数一下，超了先砍形容词和重复信息。

### ❌ 泛 vs ✓ 具体

```
中文创意：一首深夜想念异地恋人的中文流行情歌，女声，钢琴和弦乐，很戳心

太泛（❌）：a beautiful sad chinese love song, emotional, nice female voice

具体（✓）：heartfelt Mandarin pop ballad, intimate and aching, tender female
          vocals, grand piano and warm string section, slow 68 BPM, builds to
          a soaring emotional chorus, cinematic and polished
```

---

## Step 3 — 写带结构标签的歌词（≤3000 字符，语言随需求）

歌词框只放**要被唱出来的字** + **结构标签**。语言跟随用户需求（中文歌写中文词）。

- **结构骨架**：用 `[Intro] [Verse] [Pre-Chorus] [Chorus] [Bridge] [Outro]` 搭段落，**标签独占一行**。常见曲式：主歌→预副歌→副歌→主歌→副歌→桥段→副歌。
- **副歌短而重复**：1-4 行、反复出现的钩子，是全曲记忆点；主歌负责叙事（8-16 行）。
- **一行一个意思、控音节**：一行别太长，否则人声赶、咬字糊；中文一行别超过十来个字。
- **押韵自然就好**：别为押韵硬凑绕口的句子，AI 唱出来会怪。
- **人声/表演标签**：`[Whisper] [Belting] [Harmonies]` 等可点缀情绪——但要区分官方结构标签和社区扩展标签，见 `references/lyrics-and-metatags.md`。
- **纯音乐**：不写词，用 `[Instrumental]` 或段落描述（如 `[Guitar Solo]`）。

> 完整的中/英示例歌词、全量 meta tag 表、纯音乐写法，见 `references/lyrics-and-metatags.md`。

---

## Step 4 — 取标题 + 选模式/版本

- **标题**：短、有画面或记忆点，呼应主题（如《凌晨三点的时差》好过《想你》）。
- **选模式**：要精确控制就用 **Custom 模式**（Style / Lyrics / Title 分开填，本 skill 的默认假设）；只想快速碰灵感用 **Simple 模式**（一句话描述生成）。
- **版本提示**：当前主力是 **v4.5 / v5 / v5.5**（最长 8 分钟、更强提示词遵循、Personas/Voices）。按需提醒用户在生成页选当前最新主力版本。
- **再确认字符限制**：Style ≤ **240 字符**、Lyrics ≤ **3000 字符**。详见 `references/suno-features.md`。

---

## Step 5 — 调参迭代（出来不对怎么救）

第一遍不对很正常。按现象对症下药，别推倒重来：

- **先两版选一**：Suno 每次出两版，先挑底子更稳的那版再改。
- **Extend**：续写/加长，把原 Style 关键词带回去保持连贯。
- **Replace Section / Quick Replace**：只重做某一段（如副歌跑偏），省 credit。
- **Cover**：保留旋律换风格；**Persona**：复用某首的"声音气质/vibe"；**Remaster**：只精修音质不动结构。
- **Exclude styles（负面提示）**：排除不想要的乐器/风格/人声。

> 各迭代工具的适用场景、模式/版本差异、字符与时长限制，全部见 `references/suno-features.md`。

## 输出格式（交付时固定这样给）

````
🎵 **Title**：凌晨三点的时差

🎚️ **Style**（英文，复制进 Style 框 · 187/240 字符）：
```
heartfelt Mandarin pop ballad, intimate and aching, tender female vocals,
grand piano and warm strings, slow 68 BPM, builds to a soaring chorus, cinematic
```

📝 **Lyrics**（复制进 Lyrics 框）：
```
[Verse 1]
凌晨三点 手机还亮着
你那边天 是不是刚亮了
…
[Chorus]
隔着一整个时差想你
…
```

⚙️ **操作建议**：Custom 模式 · 选 v4.5/v5 · 非纯音乐 · 如不想要鼓点可在 Exclude styles 填 "drums"

💡 **说明**：锚定 Mandarin pop ballad 一个流派；"68 BPM + 钢琴弦乐"是我为戳心感补的，可调；副歌留短便于反复抓耳。
````

## 常见翻车场景与应对

| 翻车场景 | 现象 | 应对 |
|---------|------|------|
| **提示词太泛** | 出来千篇一律、塑料"AI 味" | Style 加具体乐器/BPM/人声质感/走向，别只写 `beautiful sad song` |
| **多流派打架** | 混浊、四不像、不知所云 | 只锚定一个主流派，其余用乐器/情绪/制作点缀 |
| **结构散** | 没有副歌记忆点、段落乱 | 歌词加 `[Verse]/[Chorus]/[Bridge]` 标签，标签独占一行 |
| **Style 与歌词混填** | AI 把风格描述也唱出来 / 抓不住重点 | 风格进 Style 框，要唱的字进 Lyrics 框，两边别串 |
| **人声赶、咬字糊** | 一行字太多、唱得喘 | 一行一个意思，缩短句子，控制音节 |
| **风格不对但其他都好** | 旋律喜欢、曲风想换 | 用 Cover 换风格，或 Persona 复用 vibe，别重写 |
| **混入不想要的乐器** | 出现没要的鼓/电音等 | 用 Exclude styles 负面排除，别写进 Style 描述句 |
| **字符超限** | Style 超 240 / 歌词超 3000 | 砍形容词、删重复信息，保留最有画面的词 |
| **直接点名歌手** | 写 "sing like 周杰伦" 等 | 改成描述化风格（见 style-and-vocab.md 的"像某人的感觉"） |

## 一句话总结

> 英文 Style 锚定一个流派、画好声音场景（情绪+乐器+人声+节奏+制作），结构标签搭好歌词骨架（语言随需求、一行一意），两版选一版再局部迭代——稳定出片的 Suno 歌 = 精准风格 + 各归其位的结构 + 耐心调参。
