# Midjourney 参数与语法速查

SKILL.md Step 4 调用本文件。本文件讲"MJ 的机制"——参数怎么写、取值多少、版本差异、引用图怎么用。

> ⚠️ **以 V8.1 为准（2026-06 默认模型）。** V8.1 于 2026-04-30 发布，2026-06-10 成为默认；V7 仍可手动选择。版本相关的默认值/取值可能随官方更新变动，精确数值以 docs.midjourney.com 官方文档为准。

---

## 一、语法铁律（错一个就废）

1. **参数全部放在 prompt 最末尾**，描述写完再写参数。
2. **双连字符 + 空格 + 值**：`--ar 16:9`，不是 `--ar16:9`，也不是中文破折号 `—ar`。
3. **参数之间用空格分隔**，不用逗号：`--ar 16:9 --v 8.1 --style raw`。
4. **全程英文标点**——中文逗号、中文冒号会直接让参数失效。
5. 多个参数顺序不影响结果，但习惯上 `--ar` 放最前、`--v` 紧随其后。

```
✓ 正确：a serene lake at dawn, misty mountains, soft light --ar 16:9 --v 8.1 --style raw
✗ 错误：a serene lake at dawn，misty mountains --ar16:9 —v8.1
```

## 二、核心参数表

| 参数 | 简写 | 取值范围 | 默认 | 作用 |
|------|------|---------|------|------|
| `--aspect` | `--ar` | 任意宽:高（如 1:1、16:9、9:16、2:3、4:5） | 1:1 | 画幅比例 |
| `--version` | `--v` | 8.1 / 8 / 7 / 6.1 / 6 / 5.2 … | 8.1 | 主模型版本 |
| `--niji` | — | 6 / 5 | — | 动漫/二次元专用模型 |
| `--style` | — | `raw`（V8.1/V7）；niji 另有 `cute`/`scenic`/`original`/`expressive` | 无 | `raw` = 去默认风格化，更写实更贴提示词 |
| `--stylize` | `--s` | 0–1000 | 100 | 风格化强度。低=贴提示词，高=更艺术。**V8.1 一般 ≤ 250** |
| `--chaos` | `--c` | 0–100 | 0 | 初始 4 图的差异度/多样性。越高越发散 |
| `--quality` | `--q` | .25 / .5 / 1 / 2 / 4 | 1 | 渲染精细度（吃 GPU 时间） |
| `--weird` | `--w` | 0–3000 | 0 | 怪诞/超现实程度 |
| `--seed` | — | 0–4294967295 | 随机 | 锁定初始噪声，便于复现（V8.1 同 seed+prompt 高度一致） |
| `--stop` | — | 10–100 | 100 | 在某完成度提前停，得到朦胧/未完成感 |
| `--no` | — | 任意要排除的东西 | — | 负向排除，≈ `::-0.5`。如 `--no people` |
| `--tile` | — | 开关 | 关 | 生成可无缝平铺的纹理/图案 |
| `--repeat` | `--r` | 1–40 | 1 | 同一 prompt 跑多组（批量抽卡，仅 Fast 模式） |

> **画质提示**：V8.1 的 HD 已变得又快又便宜，是默认状态；想更精细用 `--q 2`，无需旧的 `--hd` 思路。

## 三、画幅速查（--ar）

| 比例 | 用途 |
|------|------|
| `1:1` | 头像、社媒方图、Logo 概念 |
| `4:5` | Instagram 竖图、朋友圈竖图 |
| `2:3` | 人像摄影、海报、书封 |
| `9:16` | 手机壁纸、竖屏短视频首帧、Story |
| `16:9` | 桌面壁纸、横屏电影感、Banner |
| `3:2` | 经典摄影横构图、风光 |
| `4:3` | 偏方一点的经典构图 |
| `21:9` | 宽银幕、全景、电影感拉满 |

中文意图 → 比例：头像/方图→`1:1`；竖屏手机→`9:16`；海报/人像→`2:3`；桌面/电影感→`16:9`。

## 四、版本指南（怎么选 --v / --niji）

| 模型 | 特点 | 何时用 |
|------|------|--------|
| **V8.1**（默认） | 更写实、更贴提示词、出文字更准、HD 又快又便宜；偏自然语言，少堆关键词；`--s` 默认已够强 | 绝大多数情况，默认补 `--v 8.1` |
| **V7** | 默认更"艺术化"、偏好简短提示词 | 想要更强的默认风格化/绘画感时 |
| **Niji 6** | 动漫/二次元美学，线条与角色风格化 | 中文出现"动漫/二次元/日系/插画/Q版/赛璐璐"等信号 → 补 `--niji 6` |

**写实 vs 艺术**：要写实、去 AI 味 → 加 `--style raw`；要更天马行空 → 提高 `--chaos` 或 `--weird`。

## 五、多重提示与权重（::）

`::` 把一个 prompt 切成多块并各自赋权，控制谁更重要。

```
空格分隔时 space ship       → 理解成"宇宙飞船"（一个概念）
加权分隔   space:: ship     → 理解成"太空"和"船"两个独立概念
带权重     space::2 ship    → "太空"的权重是"船"的 2 倍
```

- **默认权重 = 1**；V4 以后支持小数（`::1.5`、`::0.8`）。
- **负权重**：`fruit::-0.5` 削弱某概念。所有权重之和必须为正。
- **`--no X` ≈ `X::-0.5`**，但 `--no` 更像"开关"（尽量完全排除），负权重更像"音量旋钮"（按比例削弱）。

```
serene forest::2 fog::1.5 people::-0.5 --ar 16:9 --v 8.1
```

> 一般翻译用不到 `::`；当用户强调"重点是 A，B 是陪衬"或"绝对不要 C"时才用。优先用 `--no` 处理排除。

## 六、排列组合提示（{}）

用 `{}` + 英文逗号一次生成多个变体（仅 Fast 模式）：

```
a {red, green, blue} sports car        → 生成 3 个（红/绿/蓝）
a {cat, dog} in the {jungle, desert}   → 生成 4 个（笛卡尔积）
a landscape --ar {1:1, 16:9, 9:16}     → 同一画面 3 种画幅
```

- 必须用花括号 `{}`，逗号后尽量不留空格。
- 适合用户说"给我几个不同 X 的版本"时，一条命令批量出。

## 七、引用图（图生图 / 风格 / 角色一致性）

图片 URL 放在 **prompt 最前面**（必须是公网可访问的 `.png/.jpg/.jpeg/.webp/.gif`）。

| 参数 | 配套权重 | 范围 | 作用 |
|------|---------|------|------|
| 图片 URL（图生图） | `--iw` | 0–3（默认 1） | 整体参考图：控制参考图 vs 文字的影响比重 |
| `--sref <URL>` | `--sw` | sw 0–1000（默认 100） | **风格参考**：复刻参考图的画风/色调/质感（学"怎么画"，不学"画了啥"） |
| `--cref <URL>` | `--cw` | cw 0–100（默认 100） | **角色参考**：跨图保持同一角色的脸/特征。cw 越高越严格，cw 0 只参考脸 |
| `--oref <URL>` | — | — | **Omni 参考（V7+）**：把参考图里的角色/物体放进新场景；吃 2 倍 GPU，且与 Fast/Draft/Conversational、`--q 4` 不兼容 |

```
图生图：https://example.com/ref.jpg a cozy cabin in winter --iw 1.5 --ar 16:9
风格参考：a fox in a forest --sref https://example.com/style.jpg --sw 200 --v 8.1
角色一致：a knight standing on a cliff --cref https://example.com/hero.png --cw 100
```

> 用户给了参考图链接、或要求"和这张图同风格/同一个人"时，才用本节参数；纯文生图不需要。

## 八、合规底线（PG-13）

Midjourney 内容须符合 PG-13。翻译时如遇以下中文意图，**主动引导改为合规的正向描述**，别硬译：

- 暴力/血腥/残肢、裸露/性暗示、对真实人物的诽谤或冒犯性描绘、令人不适的惊悚画面。
- 涉及真实名人姓名做敏感/诽谤画面：建议改为"某种气质/风格"的虚构人物描述。

把提示词聚焦在积极、具体、合规的视觉内容上，既合规也更稳定出图。
