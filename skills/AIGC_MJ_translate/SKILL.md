# 把中文创意翻译成精准、合规的 Midjourney 英文提示词

## 这个 skill 解决什么问题

普通翻译追求"语义对等"，但 Midjourney 吃的不是中文语义——它吃的是**视觉术语 + 正确结构 + 合法参数**。把中文直译成英文丢给 MJ，常见三种崩法：

- **直译语序**：「一个穿红裙子的女孩在雨中奔跑很有电影感」→ 直译成一句话语法，MJ 抓不住重点。
- **笼统词**：「很美」「高级感」「质感很好」翻成 `beautiful / high-end / good texture`，等于没说，MJ 只能瞎猜。
- **负面描述**：「不要有人」翻成 `no people in a negative sentence`，MJ 大概率反而画出人。

这个 skill 把"中文创意 → MJ 英文 prompt"压成**一条五步管道**：

```
Step 1  解析意图 & 按需反问   拆已知要素，关键信息缺失先问 1-2 个澄清问题
Step 2  拆解视觉要素          按八要素 checklist 归位（主体/媒介/环境/光照/色彩/情绪/构图/风格）
Step 3  精准选词翻译          查视觉术语词表，用专业英文替代笼统词
Step 4  组装结构 + 智能补参数  按推荐顺序拼装，末尾智能补 --ar / --v / --niji 等
Step 5  输出成品 + 简要说明    可复制的 prompt + 关键选词/参数说明 + 微调提示
```

> 你的角色是**视觉翻译官**，不是字典。目标不是"把每个中文词都译出来"，而是"让 MJ 画出用户脑中那张图"。

## 核心心法（贯穿全流程）

1. **翻意图，不翻字面** —— 译的是"用户想要的画面"，不是中文句子的语法结构。「治愈」不是 `healing`，是 `warm golden light, soft focus, serene atmosphere`。
2. **用具体视觉术语替代笼统词** —— `beautiful / cool / texture` 这类空话一律换成可成像的具体词（光照、镜头、材质、色板）。词表见 `references/prompt-craft.md`。
3. **只写"要什么"，不写"不要什么"** —— 正向描述。需要排除的东西用 `--no X` 参数，绝不写进描述句。
4. **八要素结构** —— 主体 + 媒介 + 环境 + 光照 + 色彩 + 情绪 + 构图/镜头 + 风格。短语 + 逗号分隔，最重要的元素放前面（语序即权重）。
5. **关键信息缺了先反问，不臆造主体** —— 主体不明 / 风格不定 / 用途比例不清时，先问 1-2 个问题，确认后再翻译。
6. **参数放结尾、空格分隔、默认 V8.1** —— 参数格式必须合法（`--param value`），不用中文标点、不漏空格。
7. **简洁但不偷懒** —— 每个词都要带来画面信息；删掉填充词，但该给的视觉细节一个不少。

## Step 1 — 解析中文意图 & 按需反问

**先别急着翻译。** 第一步是读懂用户到底想要什么，并判断信息够不够。

### 拆出已知要素

把中文描述拆进八要素的格子里，看哪些已知、哪些为空：

```
主体    谁/什么？      （女孩？猫？城市？产品？）
媒介    什么形式？      （照片？油画？3D？插画？）
环境    在哪里？        （雨夜街头？森林？纯色背景？）
光照    什么光？        （黄金时刻？霓虹？影棚光？）
色彩    什么调？        （暖色？冷青？高级灰？）
情绪    什么气质？      （治愈？暗黑？梦幻？）
构图    怎么取景？      （特写？广角？俯拍？）
风格    什么美学？      （写实？赛博朋克？吉卜力？）
用途    画幅/场景       （头像 1:1？壁纸 16:9？手机 9:16？）
```

### 关键信息缺失 → 先反问（本 skill 的硬规则）

如果出现以下任一情况，**先用 1-2 个澄清问题问清楚，确认后再翻译**，不要自己脑补：

- **主体不明确**：「画个很酷的东西」——酷的是人、车、机甲还是场景？
- **风格不定**：同一主体，写实摄影 / 动漫 / 油画出来天差地别，用户没说就问。
- **用途/画幅不清**：做头像、海报、还是手机壁纸？决定 `--ar`。

反问的方式（精简，给具体选项，别开放式追问）：

```
你想要的「赛博朋克女孩」更偏哪种？
A. 写实电影感（像《银翼杀手2049》）
B. 日系动漫（用 --niji）
C. 3D 渲染（像游戏 CG）
另外这张图用来做什么——头像(1:1) / 壁纸(16:9) / 手机(9:16)？
```

> **例外**：如果用户给的中文已经足够具体（主体、风格、画幅都清楚），或明确说了"你看着办/随便给一版"，就跳过反问，直接进 Step 2，并在 Step 5 说明哪些是 AI 推断补充的。

---

## Step 2 — 拆解视觉要素（八要素 checklist）

信息齐了，把每个要素逐一归位。这一步是"在脑子里把画面摆好"，还没到选英文词。

- **主体**：是什么 + 关键特征（数量、年龄、造型、姿态、表情）。用具体名词和具体数量（`three calico cats` 好过 `cats`）。
- **媒介**：照片 / 油画 / 水彩 / 3D 渲染 / 插画 / 雕塑……决定整体质感，必须明确。
- **环境**：地点 + 布景 + 道具 + 天气/粒子。
- **光照**：方向 + 光质 + 色温（最容易被忽略，却最决定氛围）。
- **色彩**：主色板 + 饱和度倾向。
- **情绪**：一两个氛围词。
- **构图/镜头**：景别 + 视角 + 镜头语言（焦段、景深）。
- **风格**：美学流派 / 参考艺术家或作品 / 时代。

> 八要素不必每个都填满，但**主体 + 媒介 + 光照**是地基，尽量都要有。详细的要素说明与推荐排序见 `references/prompt-craft.md`。

---

## Step 3 — 精准选词翻译

把归位好的中文要素，翻成 MJ 听得懂的专业英文术语。**这一步决定"精准"二字。**

- **查词表**：光照、镜头、色彩、材质、风格等专业术语，对照 `references/prompt-craft.md` 的中→英映射表，别用机翻直译。
- **去弱化词**：删掉 `very / really / so` 这类强度副词，换成更强的单词。`very tired` → `exhausted`；`非常大` → `colossal / gigantic`。
- **具体名词 + 具体数量**：`furniture` → `mahogany desk`；`一些鸟` → `a flock of sparrows`。
- **正向改写**：用户说「背景不要太乱」→ 不写 `not messy background`，而写 `clean minimalist background`（真要排除写进 Step 4 的 `--no`）。

举例：

```
中文：一个疲惫的女程序员深夜在工位上，氛围有点孤独，要有电影感
笼统直译（❌）：a very tired woman programmer at night, lonely, cinematic feeling
精准选词（✓）：exhausted female programmer at her desk late at night,
              glow of multiple monitors, dark empty office, single desk lamp,
              cool blue tones, melancholic and solitary mood,
              medium shot, shallow depth of field, cinematic
```

---

## Step 4 — 组装结构 + 智能补参数

### 4.1 按推荐顺序拼装

把 Step 3 的英文短语按八要素顺序串起来，**短语 + 逗号分隔**，最重要的放最前：

```
[主体 + 特征], [媒介], [环境], [光照], [色彩], [情绪], [构图/镜头], [风格]
```

### 4.2 智能补参数（本 skill 的默认行为）

根据意图**自动补上合理参数**（放在 prompt 末尾，空格分隔），并在 Step 5 逐个说明。常见映射：

| 意图信号 | 补的参数 | 理由 |
|---------|---------|------|
| 头像 / 海报 / 壁纸 / 手机屏 | `--ar 1:1` / `2:3` / `16:9` / `9:16` | 画幅匹配用途 |
| 没特别说，默认 | `--v 8.1` | 当前默认模型，最贴提示词 |
| 动漫 / 二次元 / 日系插画 | `--niji 6` | 动漫专用模型 |
| 强调写实 / 摄影 / 去 AI 味 | `--style raw` | 减少默认风格化，更贴提示词 |
| 想要更天马行空 / 多样性 | `--chaos 20~50` | 提高网格差异度 |
| 明确"不要 XX" | `--no XX` | 负向排除，别写进描述句 |

> 取值范围、语法细节、版本差异、`::` 多重权重、`{}` 排列组合、`--sref`/`--cref` 引用图，全部查 `references/mj-parameters.md`。**V8.1 默认风格化已经够强，`--s` 一般不超过 250。**

---

## Step 5 — 输出：成品 + 简要说明（本 skill 的默认输出形式）

固定用以下三段式输出：

**① 成品 prompt（代码块，方便整段复制）：**

````
```
exhausted female programmer at her desk late at night, glow of multiple monitors,
dark empty open-plan office, single warm desk lamp, cool blue and teal tones,
melancholic solitary mood, medium shot, shallow depth of field, cinematic --ar 16:9 --v 8.1
```
````

**② 简要说明（让用户看懂&能学）：**

- **关键选词**：`exhausted` 替代「很累」、`glow of multiple monitors` 把「深夜工位」的光感具体化。
- **补了哪些参数 / 为什么**：`--ar 16:9`（你说要电影感，横画幅）、`--v 8.1`（默认模型）。
- **哪些是 AI 推断补充**：「冷蓝色调、单只暖台灯」是我为了强化孤独感补的，不想要可删。

**③ 微调提示（给用户下一步的钩子）：**

- 想更竖构图发朋友圈 → 改 `--ar 4:5`
- 想完全去掉同事入镜 → 末尾加 `--no people`
- 风格太"飘"想更写实 → 加 `--style raw`

## 常见翻车场景与应对

| 翻车场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| **直译中文语序** | 把整句中文按语法译成一个英文长句 | 拆成八要素短语，逗号分隔，按权重排序 |
| **负面描述** | `no people, not blurry`（写进描述句） | 用 `--no people`；想清晰就正向写 `sharp focus` |
| **笼统词** | `beautiful, high quality, cinematic feeling` | 换成可成像的具体词：光照/镜头/色板/材质 |
| **堆砌关键词** | 把同义词堆一长串（V8.1 反而更乱） | V8.1 偏自然语言，精准短语即可，别堆 |
| **参数格式错** | `--ar16:9`、`—ar 16:9`、中文逗号 | 必须 `--ar 16:9`，双连字符 + 空格，全英文标点 |
| **臆造主体** | 用户没说清就自己脑补一个主体硬翻 | 回到 Step 1 先反问 |
| **`--s` 拉满** | 沿用旧版 `--s 750` | V8.1 默认已够，`--s` 一般 ≤ 250 |
| **真人/敏感内容** | 直接译名人姓名做诽谤/敏感画面 | MJ 是 PG-13，引导改为合规的正向描述 |

## 一句话总结

> 不要直译中文，要翻译"画面"：缺信息先反问，按八要素拆解，用专业术语精准选词，组装后智能补参数，最后给"成品 + 说明"。精准的 MJ 提示词 = 视觉术语 + 正确结构 + 合法参数，三者缺一不可。
