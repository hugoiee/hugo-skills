# AIGC 视频创作全流程引导器

## 这个 skill 解决什么问题

用户说"我想做个 AI 视频"时，他们通常处于"有模糊想法但无可执行方案"的状态。直接问"你想做什么风格"或者"给我 prompt 吧"都会卡住——前者太开放，后者太跳跃。

这个 skill 的本质是**把模糊的创作冲动结构化为一系列 10 分钟以内可决策的具体节点**，让用户在 1-3 小时对话内从零走到可执行手册。

## 核心心法（写在最前，贯穿全流程）

1. **先建框架再填内容**——框架决定上限，内容只是填满
2. **先建锚点再做扩展**——角色锚点没锁，分镜写得再细都白搭
3. **永远逼用户做选择**——"都想要"是创意死亡的开始；用三选一、二选一逼出决策
4. **永远只改一处**——迭代时一次改一处，每次输出版本号
5. **永远同步下游**——改 prompt 就要同步改难度评级、抽卡顺序、音效、速查卡
6. **永远知道什么时候停**——加东西的边际效用会递减，最后一个改动收益 < 增加复杂度时立刻停

## 七阶段总览

```
阶段 1: 项目定位      → 三问（什么 / 用途 / 时长）
阶段 2: 题材调研      → 找灵魂（这个题材区别于同类的关键点）
阶段 3: 创意收敛      → 三方向选一
阶段 4: 视觉系统锁定  → 角色 + 风格 + 画幅
阶段 5: 分镜骨架      → 三幕结构 + 每镜可解释功能
阶段 6: 细节迭代      → v1 → v2 → ... → 何时停
阶段 7: 制作准备 + Prompt 输出 → 工具适配 + 抽卡顺序
```

**关键规则：每阶段有"关键产出物"，没产出就不进下一阶段。**

## 阶段 1：项目定位

### 必答三问

用 `ask_user_input_v0` 工具一次性提出三个问题（不要一个个问，浪费用户时间）：

1. **这是什么？**
   - 选项示例：「原创概念 / 已有 IP（小说/游戏/动漫）/ 商业产品 / 品牌叙事 / 其他」
2. **核心用途？**
   - 选项示例：「投放引流 / 招商提案 / 个人作品集 / IP 涨粉 / 活动宣传」
3. **时长？**
   - 选项示例：「15-30s（信息流广告）/ 60-90s（标准宣传片）/ 2-3 分钟（深度预告）/ 还没想好」

### 为什么这三问是地基

- "什么" 决定能用什么素材、有没有版权限制
- "用途" 决定调性边界（招商片不能太作者气，个人作品反而能放飞）
- "时长" 决定信息密度上限（90s 能塞 12-15 镜头，30s 只能 4-6 个）

### 雷区

- ❌ 跳过三问直接谈创意 = 后面全部白做
- ❌ 用户给模糊答案（"嗯…就练手吧？"）——必须逼到具体
- ❌ 用户说"都行"——强迫选择

### 关键产出物

一句话项目定位，例如：
> "网文 IP 同人 / 个人作品集 / 60-90 秒"

## 阶段 2：题材调研

### 核心动作

**如果用户提到具体 IP / 主题 / 品牌：用 web_search 工具搜，不要凭印象做。**

灵魂 = 这个题材区别于同类的最关键的一点。

### 怎么找到"灵魂"

1. 搜原作/主题的"评论 / 解读 / 书评 / 用户评价"，**不要只看简介**
2. 看高赞评论里反复出现的关键词
3. 找"这个 IP/主题不像其他同类"的那个点
4. 用三个反差词描述（"暗黑 / 哲学 / 反讽" 比 "热血 / 修仙 / 长生" 准确十倍）

### 案例

- 错误做法：把"蛊真人"宣传片做成御剑飞行白衣飘飘的普通修仙
- 正确做法：先搜原作 → 发现高赞评论说"修仙界的加缪" → 抓住"加缪式存在主义 + 自封魔头反讽" → 整片调性才对得上

### 通用题材的灵魂参考方向

- 网文 IP：找"反套路点"、找"破圈关键词"
- 商业产品：找"用户最在意的痛点"，不是"卖点"
- 品牌叙事：找"品牌哲学"，不是"slogan"
- 原创概念：找"一句话最高级的卖点"

### 关键产出物

三个反差词 + 一句话灵魂描述。

## 阶段 3：创意方向收敛

### 黄金法则

**给三个真正不同的方向，逼用户做选择，不要试图融合。**

### 三方向必须真正不同

不能是同一个方向的三个变体。设计原则：

| 维度 | 方向 A | 方向 B | 方向 C |
|------|--------|--------|--------|
| 切入角度 | 必须完全不同 |
| 视觉风格 | 必须完全不同 |
| 技术难度 | 通常分高/中/低 |
| 表达类型 | 通常分叙事/视觉/概念 |

### 每个方向必须包含

1. 一句话核心概念
2. 叙事结构（按时长拆 0-15s / 15-35s / ... ）
3. 这个方向为什么"有意思"（不可替代性）
4. 适合的 AIGC 工作流（什么工具组合）

### 提问方式

用 `ask_user_input_v0`，按你心里推荐的排序：

```
options: [
  "方向 A：（具体内容）",
  "方向 B：（具体内容）",
  "方向 C：（具体内容）",
  "这三个的某种混合 / 我有别的想法"
]
```

如果用户选"混合"，**不要直接答应**——继续逼具体化，否则会变成四不像。

### 雷区

- ❌ 给三个"温和的近亲方案"
- ❌ 自己心里偏向一个但伪装中立 → 应该直接说"我推荐 X，因为……"
- ❌ 没考虑用户的工具栈和技术能力

### 关键产出物

锁定一个方向 + 一句话"为什么是这个"。

## 阶段 4：视觉系统锁定（三件套）

### 必锁的三件套

1. **角色锚点**（如果有人物出现）
2. **风格调性**
3. **画幅与构图规则**

**这三件没锁，分镜写得再细都是空中楼阁。**

### 锁定方式

用 `ask_user_input_v0` 给具体选项，不要让用户自由发挥。

#### 角色锚点的选项设计

```
- 黑衣藏脸 / 阴影遮脸（最容易保持一致性）
- 明确面部特写（难度高，表演张力强）
- 背影/侧脸/剪影为主（避免正脸，气质感拉满）
- 随剧情变化（动态设计，允许不同阶段不同造型）
```

如果用户已经有角色图，直接读图提炼角色档案表：
- 性别 / 年龄
- 发型 / 面部 / 眼神
- 服装 / 配饰
- 整体气质（用 3-5 个反差词）

#### 风格调性的选项设计

不要用形容词（"高级感""电影感"），必须用具体的参考片 / 摄影师 / 美学流派。

**详细的风格库（12 种风格 + 选择决策树）见 references/style-systems.md** — 用户要选风格时调取此文件给出 3-4 个具体选项。

#### 画幅选项

```
- 1:1（社媒友好）
- 16:9（横屏、电影感）
- 9:16（竖屏、短视频平台）
- 2.39:1（宽银幕、迪金斯感拉满）
```

### 锁定后立刻写"视觉铁律"

把三件套翻译成 5-7 条**每个镜头都必须遵守的视觉规则**。

模板：
```
| 维度 | 规则 |
|------|------|
| 光源 | 单一硬光源 + 大面积阴影 |
| 色板 | 青灰、墨蓝、铁锈红，禁用饱和色 |
| 构图 | 大量负空间，主体压在画面边缘 |
| 运镜 | 极缓慢平移 + 固定机位为主 |
| 粒子 | 必须有，但极薄（空气质感不是特效） |
| 表演 | 内敛克制，眼神戏为主 |
| 色温 | 完全偏冷，4000K 以下 |
```

并同时定义**全局负面提示词**：

```
（举例，按风格调整）
卡通风格、动漫、明亮饱和、暖橙光、柔和发光、笑容、欢快、
多光源、平光、魔法闪光、过度特效、低分辨率、模糊脸部、
畸形手指、额外手指、五官扭曲
```

### 关键产出物

角色档案表 + 视觉铁律 5-7 条 + 全局负面提示词。

## 阶段 5：分镜骨架搭建

### 黄金结构：三幕九镜

不管时长多少，先用三幕搭骨架：

```
第一幕：建立     20-30%  介绍角色与世界
第二幕：发展     40-50%  核心内容
第三幕：收尾     20-30%  情绪高潮 + 落版
```

### 不同时长的三幕分配参考

具体时长适配见 **references/narrative-structures.md**

简化版：

| 时长 | 一幕 | 二幕 | 三幕 | 镜头数 |
|------|------|------|------|--------|
| 30s | 6s | 15s | 9s | 4-6 |
| 60s | 15s | 30s | 15s | 8-10 |
| 90s | 20s | 50s | 20s | 10-13 |
| 3min | 40s | 100s | 40s | 18-25 |

### 每镜必须可解释功能

写每镜时问：**这镜如果删掉，整片会损失什么？**

如果答不上来，这镜就该删。

### 每镜必填字段

```markdown
### 镜头 NN ｜ 名称

| 项 | 内容 |
|----|------|
| 时长 | X-Ys（Zs） |
| 目的 | 这镜不可替代的功能 |
| 景别 | 特写/中景/远景/大远景 |
| 运镜 | 推/拉/摇/移/固定 |
| 节奏定位 | 动/静/冷/悠/崩（一字概括） |
| 是否用角色参考 | 是/否 |

**Prompt：**
（具体提示词）

**Plan B：**
（如果首版不满意的备选方案）
```

### 节奏曲线

第二幕不能是匀速的——必须有节奏起伏。模板：

```
动 → 静 → 冷 → 悠 → 崩
快   慢    凉    缓    重
```

这种"快-慢-凉-缓-重"的曲线才有电影感。

### 雷区

- ❌ 重复功能的镜头（两个"角色酷酷的特写"）
- ❌ 第二幕塞太多（最常见的崩盘原因）
- ❌ 没有节奏曲线（全片同一节奏）
- ❌ 关键回响镜头没设计 callback（如片头片尾的呼应）

### 关键产出物

完整分镜表，每镜功能可解释，节奏曲线有起伏。

## 阶段 6：细节迭代循环

### 黄金法则

**一次只动一处，每次输出版本号（v1 → v2 → v3...）。**

### 迭代节奏示例（来自大爱仙尊案例）

```
v1 → v2: 加结构（4 世扩到 5 世）
v2 → v3: 在一处加细节（东海加活龙）
v3 → v4: 另一处加细节（北原加狼群）
v4 → v5: 最后一处加细节（西漠加战场遗物）
```

每次改一处，但**同步所有下游**：
- 镜头 prompt
- 难度评级
- 拍摄顺序
- 音效层
- 速查卡
- 成本预估

### 提问技巧

每次迭代时，**用 ask_user_input_v0 给具体选项**：

```
❌ "你想怎么加狼？"
✅ 问 3 个具体问题：
   - 数量：1只 / 5-7只 / 10只以上
   - 颜色：黑 / 灰白 / 红眼妖兽
   - 空间关系：追逐 / 远围 / 拦截
```

### 何时停止迭代（重要）

满足任一条件就立刻停：

- [ ] 每个镜头都已有不可替代功能
- [ ] 已塞满主线 + 至少一条隐藏线
- [ ] 时长接近预设上限（90s ± 5s）
- [ ] 抽卡难度评估开始普遍升高
- [ ] 用户开始反复问"还能加什么"——这是"灵感枯竭"信号

到这一步，立刻从"加"转向"做"——开始制作。

### 雷区

- ❌ 一次改多处 → 改完不知道哪个改动出了问题
- ❌ 不出版本号 → 想回滚回不去
- ❌ 不同步下游 → 改了 prompt 但忘了改难度评级、音效
- ❌ 加到崩为止 → 90 秒装不下你想说的所有事

### 关键产出物

每个版本一份完整手册，版本号清晰，改动日志可追溯。

## 阶段 7：制作准备 + Prompt 输出

这是最后阶段，包含两件事：**生成可执行的最终 prompt** + **制作落地的全套指南**。

### 7.1 Seedance 2.0 模型介绍

本 skill 专为 **Seedance 2.0**（字节豆包系视频生成模型）设计。

**核心特点：**

- **中文友好** — 可直接用中文写 prompt，对中文语境和文化意象理解准确
- **角色一致性强** — 支持 `@图片名` 引用素材，多镜头角色锁定能力业内领先
- **物理理解到位** — 风、水、布料、光影的动力学表现自然
- **多入口灵活** — 文生视频 / 首尾帧 / 全能参考（多图 + 文本）/ 视频延长
- **运镜指令精准** — 对镜头运动术语理解清晰，可控性强
- **4–15 秒单段生成** — 默认 5s，24fps，长镜头需分段拼接

**短板与规避：**

- 高度写实人脸有概率被审核拦截 → 用风格化处理 / 背影侧脸 / 剪影规避
- 默认 16:9，2.39:1 宽银幕需后期裁切（prompt 提示主体居中）
- 音频生成不达电影级 → 抽卡阶段全程关闭，后期 Suno + 音效库替代

**API 支持参数速查（2026-04 火山方舟开放）：**

| 参数 | 取值 | 默认 | 备注 |
|------|------|------|------|
| `model` | `doubao-seedance-2-0-260128`（标准）/ `-fast-260128`（Fast） | — | Fast 版省钱、速度快、质量略低，适合抽卡 |
| `content` | 多模态数组：`text` / `image_url` / `video_url` / `audio_url` | — | 上限：图 9 张 (30MB)、视频 3 段 (50MB)、音频 3 段 (15MB) |
| `resolution` | `480p` / `720p` / `1080p` / `2K` | `1080p` | 高分辨率 token 消耗更多 |
| `ratio` | `16:9` / `9:16` / `4:3` / `3:4` / `21:9` / `1:1` / `adaptive` | `16:9` | 2.39:1 用 `21:9` 或后期裁切 |
| `duration` | 4–15（秒） | `5` |  |
| `framepersecond` | 整数 | `24` |  |
| `watermark` | bool | — | 平台水印开关 |
| `generate_audio` | bool | `false` | **抽卡阶段必须 false** |
| `return_last_frame` | bool | `false` | 长镜头拼接靠它返回末帧 |
| `camerafixed` | bool | `false` | 强制锁定机位，比 prompt 写"静止"更稳 |
| `seed` | int | 随机 | 好镜头务必记录，用于复现 |
| `callback_url` | URL | — | 任务完成回调，免轮询 |

详细参数说明与使用配方见 **references/prompt-adaptation.md**。

### Seedance 2.0 支持的运镜方式

在 prompt 里直接用中文运镜术语，Seedance 理解精准：

| 类别 | 运镜方式 | Prompt 写法 | 适用场景 |
|------|---------|-----------|---------|
| 固定 | 固定机位 | "镜头完全静止" / "固定机位" | 留白、人物特写、氛围镜 |
| 推拉 | 推镜头 | "镜头缓慢推近" / "快速推进" | 强调主体、情绪聚焦 |
| 推拉 | 拉镜头 | "镜头缓慢拉远" | 揭示环境、落幕感 |
| 横移 | 摇镜头（水平） | "镜头从左向右摇" / "横摇" | 展示空间、跟随视线 |
| 横移 | 摇镜头（垂直） | "镜头向上摇" / "俯仰摇" | 揭示高度、仰望震撼 |
| 平移 | 平移镜头 | "镜头水平平移" / "侧向移动" | 追随主体、横向叙事 |
| 跟随 | 跟拍 | "镜头跟随主体" / "跟拍" | 角色行进、动作戏 |
| 跟随 | 环绕 | "镜头围绕主体环绕一周" | 强调主体存在感 |
| 升降 | 升镜头 | "镜头上升" / "升降机上升" | 揭示规模、空灵感 |
| 升降 | 降镜头 | "镜头下降" | 沉重感、压迫感 |
| 复合 | 推 + 摇 | "镜头一边推近一边向左摇" | 复杂情绪转场 |
| 复合 | 移 + 跟 | "侧向平移跟拍主体" | 公路片、追逐戏 |
| 视角 | 第一人称 | "POV 主观镜头" | 沉浸感 |
| 视角 | 俯视 / 仰视 | "俯拍视角" / "仰拍视角" | 权力关系、心理张力 |
| 特殊 | 手持感 | "轻微手持晃动" | 纪实感、紧张感 |
| 特殊 | 希区柯克变焦 | "镜头推近的同时变焦拉远（dolly zoom）" | 心理冲击瞬间 |

**运镜书写要点：**

- 单一镜头只用 1–2 个运镜，超过会乱
- 速度词必须明确："缓慢" / "中速" / "快速"
- 起止位置可描述："从全景缓慢推至面部特写"
- 与节奏曲线对应：静态镜对应"固定机位"、崩点对应"快速推进 + 手持晃动"

详细 prompt 适配与抽卡心法见 **references/prompt-adaptation.md**

**Seedance 2.0 prompt 黄金结构：**

```
[镜头景别] + [主体描述（含角色参考）] + [主体动作]
+ [环境与背景] + [光线方向与质感] + [色调]
+ [镜头运动] + [画幅与风格]
```

**写作要点：**
- 短句 + 逗号分隔，不要写散文
- 关键元素重复 2-3 次（Seedance 会强化关注）
- 负面提示词每镜都加
- 角色参考用 `@图片1` 语法引用上传素材

### 7.2 抽卡难度评级

为每个镜头标注：

```
| 镜头 | 难度 | 抽卡次数预估 | 关键风险 |
|------|------|-------------|---------|
| 01 简单远景 | ⭐⭐ | 3-5 次 | 容易稳定 |
| 02 面部特写 | ⭐⭐⭐⭐⭐ | 10-20 次 | 人脸审核 + 一致性 |
| 03 多生物互动 | ⭐⭐⭐⭐ | 8-15 次 | 多对象一致性 |
```

### 7.3 拍摄顺序（关键策略）

**先易后难分批**：

```
第一批（先建信心）：容易稳定的镜头（远景、剪影）
第二批（建立锚点）：人脸一致性硬骨头
第三批（常规场景）：中等难度的环境镜头
第四批（视觉重点）：动作戏、特效镜头
第五批（最难硬骨头）：综合难度最高的镜头
第六批（最后扫尾）：开场镜头（往往最讲究）
```

**为什么这个顺序：** 先把容易出片的跑完，建立信心和手感；再啃硬骨头；最难的放最后那时你已经熟悉了工具脾气。

### 7.4 后期工作流（关键提醒）

**⚠️ 抽卡阶段全程关闭 AI 视频的音频生成。**

理由：AI 音频质量达不到电影级、音频绑定视频无法独立调整、浪费额度。

正确流程：
```
1. 视频抽卡（无音频）
2. 剪辑成片（无声版）
3. Suno AI 生成主音乐
4. 按音乐节奏微调剪辑切点
5. freesound / Splice / Epidemic Sound 加音效
6. DaVinci Resolve Fairlight 混音
```

**调色配方、剪辑节奏表、字体推荐等详细内容见 references/prompt-adaptation.md。**

## 跨阶段共用技巧

### 关于 ask_user_input_v0 的使用

**这个 skill 大量依赖 `ask_user_input_v0` 工具**。规则：

1. **每次最多 3 个问题** — 超过会让用户疲劳
2. **每个问题 2-4 个选项** — 太多选项等于没选项
3. **选项必须互斥** — 不能模糊重叠
4. **每个选项后加简短说明** — 用括号注明"为什么这个"
5. **加一个"让你判断"或"我有别的想法"兜底** — 避免逼用户在烂选项里选

### 关于 web_search 的使用

**任何提到具体 IP / 品牌 / 已有作品的项目，必须先 web_search**：

- 网文 IP：搜原作 + 主角名 + 剧情
- 游戏 IP：搜世界观 + 核心机制
- 品牌：搜品牌哲学 + 用户评价
- 已上线产品：搜竞品 + 用户反馈

不要凭印象做——你的训练数据可能过时或错误。

### 关于内容创作的判断

每个阶段都要"端着"——不要做用户的应声虫：

- 用户的想法不好，直接说"我不推荐，因为……"
- 用户加了一个会破坏整体的元素，直接拒绝并解释
- 用户陷入"创意疲劳"开始要求莫名其妙的细节时，主动说"我们到此为止吧，开始做"

**这个 skill 的核心价值是判断力，不是工具人。**
