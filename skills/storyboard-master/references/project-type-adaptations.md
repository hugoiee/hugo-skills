# 不同项目类型的快速适配指南

> 用法：阶段 1「项目定位」确认项目类型后，调取此文件中对应的章节，
> 按该项目类型的特殊规则调整七阶段流程。

---

## 项目类型速查表

| 类型 | 时长偏好 | 调性边界 | 重难点 |
|------|---------|---------|--------|
| 商业宣传片 | 30-90s | 不能太作者气，必须有 CTA | 卖点清晰 + 多画幅 |
| MV / 音乐宣传 | 由音乐决定 | 完全跟随音乐情绪 | 节奏卡点 |
| 个人作品集 | 60-90s | 可放飞，重技术展示 | 难度挑战 |
| 概念片 / 提案片 | 60-120s | 概念清晰 > 细节完美 | 视觉冲击 |
| IP 改编宣传片 | 60-90s | 不能离开原作精神 | 还原度 + 反差 |
| 产品演示 | 15-60s | 卖点突出 | 信息密度 |
| 品牌故事 | 60-180s | 情感优先 | 角色塑造 |
| 活动宣传 | 15-30s | 信息明确 | 视觉抓眼 |

---

## 商业宣传片

### 阶段调整要点

**阶段 1：必问**
- 投放渠道（决定画幅和时长）
- 目标受众年龄段（决定调性）
- 客户对"调性"的具体词（"高级感"要逼到具体）

**阶段 2：必查**
- 竞品的宣传片做了什么（避免撞调性）
- 客户品牌已有的视觉资产（保持一致性）

**阶段 3：方向收敛**
- 不要选"作者电影"方向
- 三个方向要分别对应：理性卖点 / 情感共鸣 / 反差冲击
- 默认推荐情感共鸣（普适性最强）

**阶段 4：视觉系统**
- 必须符合品牌已有的色彩规范
- 不能用过于实验的风格
- 字体必须用品牌字库

**阶段 5：分镜骨架**
- 第三幕必须有 CTA（产品 / 二维码 / 品牌 logo）
- CTA 时长 3-8 秒为佳
- 第二幕信息密度要高（商业片观众不会反复看）

**阶段 7：制作准备**
- 多画幅版本（16:9 横屏 + 9:16 竖屏 + 1:1 方屏）
- 多时长版本（完整版 + 15s 切片版）
- 准备 logo 动画和落版

### 商业片专属雷区

- ❌ 调性过于哲学 / 抽象
- ❌ 卖点超过 3 个
- ❌ CTA 太短或没有
- ❌ 落版没有品牌 logo

### 商业片专属心法

**"客户永远是对的"是死路。**

如果客户提的要求会让片子崩，要拿数据 / 案例反对。直接说"我们这次不能这样做，原因是……"。

## MV / 音乐宣传

### 阶段调整要点

**阶段 1：时长完全由音乐决定**
- 副歌段单独剪短版（15-30s）
- 完整版按整首歌长度

**阶段 2：必听**
- 完整音乐 3 遍
- 找到歌词的核心意象
- 记录情绪起伏的时间点

**阶段 3：方向收敛**
- 必须围绕歌曲核心情绪
- 不要做"和歌曲无关的炫技"
- 默认两条线：叙事线 + 意象线（交叠剪辑）

**阶段 4：视觉系统**
- 风格必须服务于音乐情绪
- 慢歌选缓慢风格（蒙考拉 / 塔可夫斯基）
- 快歌选动态风格（赛博 / 王家卫）
- 流行抒情可选新海诚 / 王家卫

**阶段 5：分镜骨架**
- **每个镜头的切点必须卡在节拍上**
- 副歌段必须有视觉高潮
- 间奏部分用空镜或意象镜头

**阶段 7：制作准备**
- 音乐先有，按音乐节奏剪辑
- 镜头时长精确到 0.1 秒
- 不需要太多镜头，重质不重量

### MV 专属雷区

- ❌ 镜头切点和节拍不同步
- ❌ 视觉高潮不在副歌
- ❌ 风格和音乐情绪冲突

## 个人作品集

### 阶段调整要点

**阶段 1：时长**
- 推荐 60-90s（太短不够展示，太长用户划走）

**阶段 3：方向收敛**
- 可以选最难的方向
- 重点是展示"你能驾驭什么别人做不了的"
- 默认推荐"作者电影"方向

**阶段 4：视觉系统**
- 选你最擅长的风格（不要追新潮）
- 一定要有"个人签名"（一个其他作品里不会出现的视觉元素）

**阶段 5：分镜骨架**
- 可以塞作者表达
- 隐藏叙事线可以多条（懂的观众会发现）
- 镜头难度可以拉满

**阶段 6：细节迭代**
- 可以多迭代几版（不赶时间）
- 但同样要知道何时停（不要陷入完美主义）

**阶段 7：制作准备**
- 重点是"展示技术能力"，挑战难镜头
- 后期处理要做到位（调色、字幕、音乐都不能糊）
- 输出 4K（HD 不够档次）

### 作品集专属雷区

- ❌ 选了不擅长的风格
- ❌ 没有个人签名 → 看完不记得是谁的
- ❌ 技术上太保守 → 看完没记忆点

### 作品集专属心法

**作品集的目的是"被记住"，不是"完美"。**

宁可有 1 个让人惊艳的瞬间 + 9 个普通镜头，
不要 10 个都"还行"的镜头。

## 概念片 / 提案片

### 阶段调整要点

**阶段 1：核心问题**
- 提案给谁看？（投资人 / 客户 / 大众）
- 提案的核心论点是什么？

**阶段 2：必查**
- 同类概念已有什么作品（避免雷同）
- 受众群体的接受度

**阶段 3：方向收敛**
- 概念清晰 > 细节完美
- 必须能用一句话说清楚"这片在讲什么"

**阶段 4：视觉系统**
- 可以放弃严格的角色一致性（人物可以不出现 / 或剪影）
- 重视觉冲击 > 重叙事完整

**阶段 5：分镜骨架**
- 镜头数可以少（重点是概念清晰）
- 每个镜头都要"为概念服务"

**阶段 7：制作准备**
- 重点是 Midjourney 出图 + Seedance 2.0 简单转视频
- 不需要复杂特效
- 字幕可以多用（强化概念表达）

### 概念片专属心法

**它不需要是"完整作品"，它是"想象的种子"。**

让观众看完后脑补"如果做成完整作品会有多酷"——这就成了。

## IP 改编宣传片

### 阶段调整要点

**阶段 2：必须做的题材调研**
- 阅读原作 / 玩过游戏 / 看过原版
- 找到原作的"灵魂"（不是表象）
- 收集粉丝最关心的元素

**阶段 3：方向收敛**
- 不能脱离原作精神（粉丝会骂）
- 但要做"反预期的还原"（不要做粉丝已经能想到的东西）
- 默认推荐"原作核心母题的极致提纯"

**阶段 4：视觉系统**
- 风格选择要考虑原作的气质
- 角色形象必须经得起粉丝考验
- 不要做"美化版"（粉丝最讨厌）

**阶段 5：分镜骨架**
- 可以塞原作的"经典符号"作为彩蛋
- 不要直接复刻原作镜头（除非这是目的）
- 落版可以用原作的标志性元素

### IP 改编雷区

- ❌ 把原作做成"通用模板"（如把蛊真人做成普通修仙）
- ❌ 加了原作里没有的元素（破坏世界观）
- ❌ 角色形象和原作差太远

### IP 改编心法

**最高境界：粉丝看完说"这就是我心里的样子"。**

要达到这个，必须真的理解原作。**绝对不要凭印象做 IP。**

---

## 产品演示

### 阶段调整要点

**阶段 1：核心问题**
- 这个产品最大的卖点是什么（必须一个，不能多）
- 用户最关心的痛点是什么

**阶段 3：方向收敛**
- 三个方向分别对应：功能展示 / 使用场景 / 情感共鸣
- 默认推荐使用场景（最接地气）

**阶段 4：视觉系统**
- 产品本身就是"角色"
- 风格不能盖过产品（除非品牌就是风格驱动）

**阶段 5：分镜骨架**
- 产品出现的镜头要清晰（不能虚化）
- 卖点演示镜头要足够长（观众要看清楚）
- 落版必须有产品 + 名字

### 产品演示雷区

- ❌ 卖点过多 → 一个也记不住
- ❌ 产品镜头一闪而过
- ❌ 落版没产品名字

---

## 品牌故事

### 阶段调整要点

**阶段 1：核心问题**
- 品牌的"为什么"（不是"是什么"）

**阶段 2：必查**
- 品牌的真实历史
- 创始人故事
- 用户的真实评价

**阶段 3：方向收敛**
- 必须有"主角"（创始人 / 用户 / 拟人化品牌）
- 必须有"转折"（不能全是好事）
- 三个方向：创业奋斗 / 用户故事 / 文化传承

**阶段 4：视觉系统**
- 温暖调性优先（除非品牌就是冷感）
- 自然光为主，不要太电影感

**阶段 5：分镜骨架**
- 第一幕建立人物
- 第二幕必须有低谷 / 困难
- 第三幕情感升华 + 品牌出现

### 品牌故事心法

**品牌出现得越晚越好。**

不要一开头就 logo，最好在 60% 时间后才让观众意识到"原来是某品牌"。

---

## 活动宣传

### 阶段调整要点

**阶段 1：必问**
- 活动核心信息（时间、地点、谁参加）
- 推广目的（报名 / 关注 / 现场到达）

**阶段 3：方向收敛**
- 信息明确 > 风格新颖
- 默认推荐"信息密度高 + 视觉抓眼"

**阶段 4：视觉系统**
- 一切服务于"让人记住活动名字"
- 字幕 / 海报感强
- 不要太电影感（影响信息传达）

**阶段 5：分镜骨架**
- 活动名字必须出现 2 次以上
- 时间地点必须清晰展示
- 嘉宾 / 卖点单独镜头

### 活动宣传雷区

- ❌ 太电影感导致信息淹没
- ❌ 活动名字只出现一次
- ❌ 时间地点不清晰

---

## 通用规则：所有项目都要遵守的

### 1. 不管什么类型，都要做完整的题材调研

凭印象做项目 = 灾难的开始。

### 2. 不管什么类型，都要锁定三件套

角色 + 风格 + 画幅，没锁就不进分镜阶段。

### 3. 不管什么类型，都要有 callback

短片不复杂，但必须"完整"。callback 是完整性的来源。

### 4. 不管什么类型，都要遵守"一次只改一处"

迭代时控制变量，否则失控。

### 5. 不管什么类型，都要知道何时停

加东西的边际效用会递减。该停就停。

## 项目类型识别提示

如果用户没明说项目类型，但出现以下关键词，可以判断：

| 关键词 | 项目类型 |
|--------|---------|
| "买"、"购买"、"产品"、"卖" | 商业 / 产品 |
| "歌"、"音乐"、"MV"、"歌手" | MV |
| "作品集"、"练手"、"个人作品" | 个人作品集 |
| "提案"、"投资人"、"内部演示" | 提案片 |
| "粉丝"、"IP"、"小说"、"游戏" | IP 改编 |
| "品牌"、"我们的故事"、"创始人" | 品牌故事 |
| "活动"、"发布会"、"展会" | 活动宣传 |
| "概念"、"未来"、"想象" | 概念片 |
