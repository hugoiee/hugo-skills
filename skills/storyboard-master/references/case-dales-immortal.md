# 完整案例：大爱仙尊宣传片项目

> 这是这个 skill 的"工作样板"。新项目第一次跑时，先扫一遍这个案例，
> 能快速理解七阶段如何串起来，以及每阶段实际的对话节奏。

---

## 项目基本信息

- **题材：** 网文 IP《蛊真人》同人宣传片
- **用户身份：** AIGC 视频学习者，正在做个人作品集
- **总耗时：** 约 2 小时对话
- **总产出：** 5 个版本的完整制作手册（v1 → v5）
- **最终成果：** 13 镜 90 秒方案，完全可执行

---

## 七阶段实际节点回顾

### 阶段 1：项目定位（耗时 5 分钟）

**Claude 问：**
```
ask_user_input_v0 三问：
- 「大爱仙尊」是指什么？
- 宣传片核心用途？
- 时长大概多久？
```

**用户答：**
- 网文/小说 IP
- 纯粹个人作品 / 练手
- 60-90 秒（标准宣传片）

**关键产出：** "网文 IP 同人 / 个人作品 / 60-90 秒"

**这一阶段的得失：**
- ✅ 一次性问完三个核心问题，效率高
- ✅ 给具体选项，用户不用思考太久

---

### 阶段 2：题材调研（耗时 10 分钟）

**Claude 动作：**
- 用 web_search 搜"大爱仙尊 小说 主角 剧情简介"
- 发现这是《蛊真人》主角自封尊号
- 看高赞评论发现"修仙界的加缪"评语
- 总结出三个反差词：**暗黑 / 哲学 / 反讽**

**关键产出：**
> "加缪式存在主义 + 自封魔头反讽 + 蛊虫暗黑美学"

**这一阶段的得失：**
- ✅ 没凭印象做（如果凭印象，会做成普通修仙）
- ✅ 找到了原作独有的"加缪"气质 — 后续整片都基于这个

---

### 阶段 3：创意方向收敛（耗时 15 分钟）

**Claude 给出三个真正不同的方向：**

| 方向 | 核心 | 难度 | 类型 |
|------|------|------|------|
| A 自封 | 一场加冕的反讽 | 中 | 叙事型 |
| B 蛊 | 把"蛊"做成视觉主角 | 低（最简单） | 视觉型 |
| C 长生 | 西西弗斯式抗争 | 高（最难） | 作者电影型 |

**用户选：方向 C（西西弗斯式长生执念）**

**关键产出：** 锁定方向 C — "他每一世都浴血归来，每一世都死了重来"

**这一阶段的得失：**
- ✅ 三个方向真正不同（不是同一方向的变体）
- ✅ 用户选了最难的方向，符合个人作品集的特性
- ✅ Claude 没装中立，明确推荐了 C

---

### 阶段 4：视觉系统锁定（耗时 15 分钟）

**Claude 用 ask_user_input_v0 一次问三件套：**

```
问 1：主角方源的视觉设定？
  - 黑衣藏脸（最稳）
  - 黑衣+面部特写（难度高）← 用户选这个
  - 背影/侧脸为主
  - 随剧情变化

问 2：轮回的视觉表现？
  - 环境剧变 ← 用户选这个
  - 服装剧变
  - 状态剧变
  - 纯粹象征

问 3：画质调性？
  - 沙丘古黑
  - 蒙考拉冷玩 ← 用户选这个
  - 栖地东方幻
  - 辛麦贩赛博
```

**用户后来还提供了角色图**（这是关键加成）

**Claude 提炼角色档案：**
- 长黑发湿润凌乱
- 墨绿血衣 + 葫芦腰挂
- 极冷白皮肤 + 飞溅血迹
- 空洞冷静眼神
- 三个反差词：**血道修士 / 渡劫归来 / 杀伐过重**

**Claude 立刻写视觉铁律：**
（蒙考拉冷玩的 7 条铁律，参见 style-systems.md）

**关键产出：** 角色档案 + 视觉铁律 + 全局负面提示词

---

### 阶段 5：分镜骨架（耗时 30 分钟）

**Claude 用三幕结构搭骨架：**

```
第一幕「执念」  0-25s   3 镜（春秋蝉 / 面部锚定 / 悬崖独立）
第二幕「轮回」  25-58s  4 镜（南疆 / 西漠 / 北原 / 废墟蛊群）
第三幕「自封」  58-90s  3 镜（面部回响 / 虚空自封 / 落版）
```

每镜都填了完整字段（时长、目的、景别、运镜、节奏定位、是否用角色参考、**验收标准**）。

**单镜样例（镜头 07 东海活龙）：**

| 项 | 内容 |
|----|------|
| 时长 | 45-54s（9s，需分段拼接） |
| 目的 | 神兽敬畏 — 这镜删掉，"凌驾于天地之灵"的哲学就立不住 |
| 景别 | 中景 → 面部特写（推进） |
| 运镜 | 缓慢推进 + 主体环绕 |
| 节奏定位 | 悠 |
| 角色参考 | 是（@主角图1） |

**验收标准：**
- [ ] 主角面部与角色档案表一致（银白长发、青灰瞳色）
- [ ] 龙为活龙，鳞片有光泽，眼神清醒（**不得**出现死龙状态——那是镜头 08）
- [ ] 主角与龙形成"对视"关系，视线高度匹配
- [ ] 光源为单一冷色顶光，海面有反光
- [ ] 色温 < 4000K，墨蓝与青灰主导
- [ ] 镜头在 9s 内完成从中景到面部特写的推进，无停顿
- [ ] 一票否决：无暖光、无笑容、无卡通龙感、龙鳞无畸变

**关键产出：** v1 分镜表 10 镜，输出为完整 markdown 手册（每镜含 prompt、Plan B、验收标准）。

---

### 阶段 6：细节迭代（耗时 60 分钟）

这是最长的一段，也是最有价值的一段。

#### v1 → v2：结构扩展

**用户要求：** 加入东海和中州（蛊真人五域设定）

**Claude 处理：**
- 把四世扩到五世
- 重新分配时间预算（第二幕从 33s 扩到 52s）
- 设计五域的视觉差异（材质、温度、空间感）
- 设计五域出场节奏（动-静-冷-悠-崩）

**关键决策：** 中州放最后（"文明崩塌作为轮回收尾"）

#### v2 → v3：细节加深 1

**用户要求：** 东海加入龙

**Claude 处理：**
- 先讨论龙的设计决策（颜色、数量、空间关系）
- **推动用户选"对视"方案**（戏剧张力最强）
- **同步调整中州为"死龙残骸"**（形成"活龙仰望 / 死龙俯卧"的呼应）
- 设计 07→08 的"眼神匹配剪辑"

**关键决策：** 视觉哲学升级 — "他凌驾于天地之灵之上"

#### v3 → v4：细节加深 2

**用户要求：** 北原加入狼群

**Claude 处理：**
- 设计黑色巨狼（不是普通灰狼）
- 设计头狼"哀祭"而非"追猎"
- 把生灵态度链补完整：南疆敌意 → 北原畏惧 → 东海敬畏 → 中州被超越

#### v4 → v5：细节加深 3

**用户要求：** 西漠加入元素

**Claude 处理：**
- **质疑用户的需求**："西漠的沉默是有功能的，加东西会破坏节奏"
- 提供三个不同程度的方案（A 加遗物 / B 拆两镜 / C 加生灵）
- 推荐方案 A（保留沉默功能，但加战场遗物增加视觉密度）
- 同步更新生灵态度链："沉默"改为"屠尽"

**这一阶段最重要的得失：**
- ✅ 每次围绕一个改动主题，但主题内的所有相关项一次性改完（如 v2→v3 加活龙时，同步把中州改为死龙残骸形成呼应；v4→v5 加遗物时，同步把生灵态度链的"沉默"改为"屠尽"）
- ✅ 每次都同步下游（剪辑表、难度评级、音效、速查卡）
- ✅ 每次都出版本号（v1 → v5）
- ✅ Claude 适时质疑用户需求（不做应声虫）

#### 何时停止迭代

到 v5 后，Claude 主动建议："**到这里打住，开始动手做**"。

理由：
- 13 镜每个都已有不可替代功能
- 已塞满主线 + 一条完整的隐藏叙事线（生灵态度链）
- 时长接近 90s 上限
- 抽卡难度评估开始普遍升高

**这是这个 skill 最重要的判断之一**：知道什么时候停。

## 这个项目的关键学习点

### 学习点 1：题材调研决定上限

如果阶段 2 没搜原作，把"大爱仙尊"做成普通修仙片——后面再细致也是错的。

**先搜，再做。**

### 学习点 2：三件套必须先锁

角色没锁、风格没锁、画幅没锁，分镜写得再细都是空中楼阁。

**地基没打稳，楼盖得越高摔得越惨。**

### 学习点 3：迭代时一次一个主题，但主题内全改透

v1 → v5 五个版本，每个版本围绕一个改动主题，但把主题内的所有相关项 + 所有下游一次性改完，不留半成品。

例如 v2→v3 加"东海活龙"这个主题时，不只是改东海镜的 prompt，还同步把中州改成"死龙残骸"形成呼应，并设计眼神匹配剪辑——整个主题相关的牵连项一次性改透。

**主题维度上一次一个（便于归因），主题内部全部改透（避免脱节）。**

### 学习点 4：要敢于质疑用户

v5 阶段用户要求"西漠加东西"，Claude 没有立刻照做，而是先质疑"西漠的沉默有功能"，再给三个方案让用户选。

**这个 skill 的核心价值是判断力，不是工具人。**

### 学习点 5：要知道何时停

最难的不是"加东西"，是"停下来"。

到 v5 时所有镜头都有不可替代功能、隐藏线齐全、节奏密度饱和——
Claude 主动说"打住，开始做"。

**加到崩之前先停。**

## 对话节奏样本

这是阶段 1 实际对话的样本（简化版）：

```
用户：我要使用AIGC生成视频技术，为大爱仙尊做一部宣传片，
     但是现在还没有思路

Claude：好的，宣传片这种东西最怕一开始就闷头写脚本，
       先把方向定下来再展开会顺很多。
       
       [使用 ask_user_input_v0 一次性问三个核心问题]

用户：[在 UI 里点选答案]
- 网文/小说 IP
- 纯粹个人作品/练手  
- 60-90 秒

Claude：明白了，那这个项目的定位就清楚了——
       网文 IP 同人向宣传片，60-90 秒，作品集导向。
       
       这个定位其实给了你最大的创作自由度：
       不用迁就版权方调性，不用考虑投放转化，
       可以把它当成"AIGC 视频工作流的技术 demo + 美学表达"来做。
       
       在给你具体创意方向之前，我先做点功课——
       我对"大爱仙尊"这部小说不熟，得先搞清楚核心设定
       才能给你出有针对性的方向，不然就是空对空。
       
       [使用 web_search 搜索原作]
```

**节奏要点：**
- 不说废话
- 用问题驱动而不是描述
- 在做工具调用前先告诉用户为什么要做
- 给阶段性结论（"这个定位的好处是……"）

## 这个案例告诉新项目的事

1. **复杂项目不复杂**——只要按七阶段走，2 小时内能从零到完整方案
2. **细节不是凭空加的**——每个细节都要服务于核心母题和隐藏线
3. **判断力比想象力重要**——知道什么时候停，知道什么时候反对用户
4. **工程化创意是可能的**——把"灵感"拆成"可决策的具体节点"

如果新项目卡在某个阶段，可以回来看这个案例对应的阶段是怎么处理的。
