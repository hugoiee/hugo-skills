# AIGC 视频四步管道（Storyboard → Image → Video）

## 这个 skill 解决什么问题

用户说"我想做个 AI 视频"时，他们通常拿不出可执行的 prompt。直接问"你想要什么风格"会卡住，直接给 prompt 又是无源之水。

这个 skill 把流程压成**一条四步管道**：

```
Step 1  创意收敛       一次 AskUserQuestion 锁死题材 / 用途 / 时长
Step 2  分镜手册       三幕骨架 + 视觉三件套 + 每镜验收标准
Step 3  出图 prompt    ChatGPT-Images 2 单帧 prompt（含角色一致性策略）
Step 4  视频 prompt    Seedance 2.0 首帧+文 模式 + 参数清单
```

最终交付物是 **一份按镜头组织的 markdown 工作手册**，每镜下方分别给出图 prompt 和视频 prompt，用户复制粘贴即可执行。Skill 本身不调用任何 API——所有外部操作（ChatGPT 网页出图、火山方舟生成视频）由用户手动完成并把结果文件名回填到手册。

## 核心心法（贯穿全流程）

1. **图与视频 prompt 分离** —— 图像 prompt 写"这一帧长什么样"，视频 prompt 写"在这一帧基础上动起来后是什么"。两者职责完全不同，混写会两边都崩。
2. **永远逼用户做选择** —— "都想要"是创意死亡的开始；用三选一 / 二选一 + "我有别的想法"兜底。
3. **每镜必有验收标准** —— 不写验收标准 = 抽卡时全凭感觉，浪费额度。
4. **主题内全改透** —— 每次迭代围绕一个主题，但主题内所有相关镜头、prompt、参数一次性同步，不留半成品；输出版本号 v1 → v2。
5. **知道什么时候停** —— 边际效用 < 复杂度增加时立刻停，从"加"转"做"。

## Step 1 — 创意收敛

### 关键动作

用 `AskUserQuestion` **一次性**问三件事（不要一个个问）：

1. **题材类型**：原创概念 / 已有 IP（小说、游戏、动漫）/ 商业产品 / 品牌叙事 / 其他
2. **核心用途**：投放引流 / 招商提案 / 个人作品集 / IP 涨粉 / 活动宣传
3. **时长**：15–30s（信息流）/ 60–90s（标准宣传片）/ 2–3 分钟（深度预告）

**如果题材是已有 IP / 品牌 / 已上线产品，立刻 `WebSearch`** 搜原作 + 高赞评论 + 用户解读，找"灵魂"——这个题材区别于同类的最关键的一点（用三个反差词描述，比如"暗黑/哲学/反讽"比"热血/修仙/长生"准确十倍）。**不要凭印象做。**

### 雷区

- ❌ 跳过三问直接谈创意 → 后面全部白做
- ❌ 接受"嗯…就练手吧"这种模糊答案 → 必须逼到具体
- ❌ 跳过题材调研直接写分镜 → 大概率做歪

### 产出物

一句话项目定位 + 三个反差词。例：
> "网文 IP 同人 / 个人作品集 / 60–90s · 暗黑 + 哲学 + 反讽"

## Step 2 — 分镜手册生成

### 2.1 视觉三件套（必锁，三件没锁分镜白写）

用 `AskUserQuestion` 给具体选项锁死：

**① 角色锚点**（如有人物）
- 黑衣藏脸 / 阴影遮脸（一致性最容易）
- 明确面部特写（难度高、表演张力强）
- 背影 / 侧脸 / 剪影为主（气质感拉满）
- 随剧情变化（动态设计）

如果用户已经有角色图，直接读图提炼角色档案表：性别 / 年龄 / 发型 / 面部 / 眼神 / 服装 / 配饰 / 3–5 个气质反差词。

**② 风格调性** —— 调 `references/style-quickpick.md`，给用户 3–4 个具体选项（带参考片名和色板），禁止用"高级感""电影感"这种空话。

**③ 画幅**
- `1:1`（社媒）/ `16:9`（横屏电影感）/ `9:16`（竖屏）/ `21:9`（宽银幕迪金斯感）

### 2.2 视觉铁律（锁完三件套立刻写）

把三件套翻译成 5–7 条**每个镜头都必须遵守**的规则。模板：

| 维度 | 规则示例 |
|------|---------|
| 光源 | 单一硬光源 + 大面积阴影 |
| 色板 | 青灰、墨蓝、铁锈红，禁用饱和色 |
| 构图 | 大量负空间，主体压在画面边缘 |
| 运镜 | 极缓慢平移 + 固定机位为主 |
| 表演 | 内敛克制，眼神戏为主 |
| 色温 | 完全偏冷，4000K 以下 |

并定义**全局负面提示词**（出图 + 视频两边都用）：
> 卡通、动漫、明亮饱和、暖橙光、笑容、欢快、平光、多光源、低分辨率、模糊脸部、畸形手指、额外手指、五官扭曲

### 2.3 三幕骨架（按时长 + 节奏意图共同决定）

**三幕时长分配是骨架（相对稳定），但镜头数完全由用户的节奏意图决定，不要写死范围。**

| 时长 | 一幕（建立） | 二幕（发展） | 三幕（收尾） |
|------|-------------|-------------|-------------|
| 30s  | 6s  | 15s  | 9s  |
| 60s  | 15s | 30s  | 15s |
| 90s  | 20s | 50s  | 20s |
| 3min | 40s | 100s | 40s |

**镜头数怎么定**：

在写分镜前**先问用户节奏意图**（用 `AskUserQuestion`），常见三档：

| 节奏意图 | 单镜均时 | 适用 | 示例 |
|---------|---------|------|------|
| **快剪 / 高密度** | 0.5–1.5s | 信息流广告、燃向 MV、产品卖点轰炸 | 15s 可装 10–20+ 镜 |
| **标准叙事** | 2–5s | 宣传片、品牌叙事、剧情向短片 | 60s 约 12–20 镜 |
| **慢叙事 / 氛围向** | 5–15s | 概念片、艺术短片、长镜头美学 | 90s 可能只有 6–10 镜 |
| **混合** | 视幕而定 | 大部分项目 | 一幕慢、二幕快、三幕收 |

**原则**：
- **不给硬性镜头数范围**——同一时长，快剪和慢叙事的镜头数可以差 5–10 倍
- 长镜头（单镜 > 5s）是合法选择，**不要因为"镜头太少"就强行拆**
- 镜头数推导链：**用户节奏意图 → 单镜均时 → 镜头数 = 总时长 / 单镜均时**
- 用户若说不出节奏意图，给出 3 个具体示例（"像华为广告那种快剪 / 像苹果广告那种标准叙事 / 像王家卫那种慢"）让其选

第二幕节奏曲线必须起伏：`动 → 静 → 冷 → 悠 → 崩`（快-慢-凉-缓-重），全片同节奏 = 无电影感。即使整片是快剪，二幕里也要插 1–2 个慢镜做呼吸点。

### 2.4 每镜必填字段

```markdown
### 镜头 NN ｜ 名称

| 项 | 内容 |
|----|------|
| 时长 | X–Y s（Z s） |
| 镜头功能 | 这镜不可替代的功能（删掉会损失什么） |
| 景别 | 特写 / 中景 / 远景 / 大远景 |
| 运镜 | 推 / 拉 / 摇 / 移 / 跟 / 固定 |
| 节奏定位 | 动 / 静 / 冷 / 悠 / 崩 |
| 是否用角色参考 | 是 / 否 |
```

### 2.5 验收标准（每镜 4–6 条，必须可二元判断）

| 类别 | 模板 | 示例 |
|------|------|------|
| 一致性 | 角色 X 的 Y 必须与档案表一致 | 主角发型为银白长发、瞳色为青灰 |
| 视觉铁律 | 必须符合视觉铁律第 N 条 | 单一硬光源，无多光源平光 |
| 技术规格 | X 必须为 Y | 画幅 21:9、时长 5±1s、24fps |
| 运镜匹配 | 运镜与节奏定位一致 | 节奏"崩"对应"快速推进+手持晃动" |
| 功能可解释 | 镜头核心信息必须传达 | 观众能读出"角色凌驾于天地之灵之上" |
| 一票否决 | 不得出现 X | 不得出现笑容、暖光、卡通感、畸形手指 |

**坏例子**：「画面要有电影感」「色调要对」「角色要帅」—— 全部不可判断，删掉。

### 产出物

完整分镜表（每镜含上述字段 + 验收标准）+ 视觉铁律 5–7 条 + 全局负面词。

## Step 3 — 图像生成（ChatGPT-Images 2）

### 核心原则

**图像 prompt 只描述"这一帧静止时长什么样"**，不要写动作、不要写运镜。动作和运镜留给 Step 4。

### ⚠️ 12 镜分册硬限制

**ChatGPT-Images 2 单次对话最多承载 12 镜的出图任务**。理由：超过 12 镜后，上下文中的角色档案图会被稀释、风格漂移显著加剧、参考图复用记忆退化、ChatGPT 容易丢失早期已锁定的视觉细节。

**规则：**

- 镜头数 ≤ 12：一份故事版（一个 ChatGPT 对话）跑完
- 镜头数 > 12：**必须分册**——每册 ≤ 12 镜，每册独立开一个 ChatGPT 对话
- 工作手册顶部明确标注分册方案，例如 18 镜项目分为「册一：镜 01–10（第一幕 + 第二幕前半）」「册二：镜 11–18（第二幕后半 + 第三幕）」
- **分册切割点优先选幕之间或场景之间**，不要从一个连续动作中间切开
- 每册都要**完整重新上传** `character-ref.png` + 视觉铁律摘要 + 全局负面词，不能依赖"上一册里我们说过……"

**分册时新对话的开场模板**：

```
项目：[一句话定位]
本对话负责镜头 [N–M] / 共 [总数] 镜（第 [X] 册）。

附件：character-ref.png（角色档案，所有镜头保持一致）

视觉铁律（每镜必守）：
1. ...
2. ...
（5–7 条）

全局负面词：...

接下来我会依次发出每镜的 prompt，请基于参考图保持角色和风格一致。
```

如果 Step 2 的分镜表超过 12 镜，**在写出图 prompt 之前就先输出分册方案**给用户确认。

### 出图 prompt 黄金结构

```
[景别] + [主体造型/服装/表情/姿态]
+ [环境布景与道具] + [光源方向与质感]
+ [色板 / 色温] + [画幅 + 风格关键词]
+ Negative: [全局负面词 + 本镜专属负面词]
```

短句 + 逗号分隔，关键元素（色板、光源、风格）重复 2–3 次强化。

### 角色一致性策略（核心痛点）

**第一步**：在 ChatGPT 里先生成一张"**角色档案图**"——正面 + 侧面 + 背面三视图，或者一张高质量肖像作为锚点。存为 `character-ref.png`。

**第二步**：每镜出图时，在 ChatGPT 对话中 **上传 `character-ref.png` 作为参考**，文字 prompt 里写"基于参考图保持角色一致：[本镜的造型描述]"。

**第三步**：每镜的成图存为 `shot-NN.png`，回填到工作手册对应镜头。

详细出图配方（构图术语词典、5 种镜头类型模板、避坑清单）见 `references/image-prompt-recipes.md`。

### 产出物

每镜一段出图 prompt（含本镜专属负面词）+ 一张 `character-ref.png` + 每镜一张 `shot-NN.png`。

## Step 4 — 视频生成（Seedance 2.0）

### 核心原则

**用"首帧图 + 运动描述"模式**（火山方舟"首帧+文"入口），把 Step 3 的 `shot-NN.png` 作为首帧，prompt 里只写"在这一帧基础上发生了什么 + 镜头怎么动"。这是最稳的入口，比纯文生视频一致性高一个量级。

### 视频 prompt 黄金结构

```
首帧：shot-NN.png（手动上传）

文字 prompt：
[主体动作（1–2 句短句）] + [镜头运动术语（单一或简单复合）]
+ [节奏副词：缓慢/中速/快速] + [氛围一句话]
+ Negative: [全局负面词]
```

### 关键参数（每镜必标）

```
duration: 5            # 4–15s，默认 5
ratio: 21:9            # 与画幅一致
resolution: 1080p      # 抽卡用 720p 省钱
generate_audio: false  # 抽卡阶段必须 false
camerafixed: true      # 静态镜务必开
seed: ___              # 出好镜后记录用于复现
```

完整 API 参数表 + 运镜术语表 + 抽卡策略见 `references/video-prompt-recipes.md`。

### 抽卡顺序（先易后难，建立信心）

```
第 1 批：远景 / 剪影（容易稳定）
第 2 批：人脸一致性硬骨头
第 3 批：中等难度环境镜
第 4 批：动作戏 / 多对象互动
第 5 批：开场镜头（往往最讲究，放最后做时手感最熟）
```

### 抽卡阶段全程关闭音频

AI 音频达不到电影级，且绑定视频无法独立调整。正确流程：
1. 视频抽卡（无音频）→ 2. 剪辑无声版 → 3. Suno 生成主音乐 → 4. 按音乐微调切点 → 5. freesound / Epidemic Sound 加音效 → 6. DaVinci Fairlight 混音。

### 产出物

每镜的视频 prompt + 参数块 + 抽卡难度评级（⭐ 1–5）+ 预估抽卡次数。

## 跨步通用准则

### AskUserQuestion 规则

- 每次最多 3 个问题，每个 2–4 个选项
- 选项必须互斥，每个后面用括号注明"为什么这个"
- 永远加"我有别的想法 / 让你判断"兜底

### 何时停止迭代

满足任一条件立刻停，转入抽卡：
- 每镜都有不可替代功能
- 时长接近预设上限
- 用户开始反复问"还能加什么"（灵感枯竭信号）
- 抽卡难度评估开始普遍 ⭐⭐⭐⭐ 以上

### 端着，不做应声虫

- 用户的想法不好直接说"我不推荐，因为……"
- 用户加破坏整体的元素，直接拒绝
- 用户陷入创意疲劳要求莫名其妙的细节时，主动说"到此为止，开始做"

**这个 skill 的核心价值是判断力，不是工具人。**
