# 单人口播视频（口播稿 → 切段卡时长 → 逐段说话 prompt）

> 写口播稿 → 按语义切成偏 10–15s 的段、卡死总时长 → 锁一张单人固定机位首帧 → 逐段出**干净可直接复制**的说话 prompt（连贯讲话、不发呆）

## 这个 skill 解决什么问题

用户手写好了单人口播稿（一个人对着镜头讲），想用即梦 / 可灵 / Vidu 逐段生成视频。直接一句一镜、或随手设个时长，会踩三个坑：

常见三种崩法：

- ❌ **一句一镜、切太碎** —— 切出十几个三五秒的小片段，拼起来又碎又跳，剪辑也累。
- ❌ **台词短、时长设长 → 人物"傻坐着发呆"** —— 生成时长比台词长，模型就拿发呆 / 愣神 / 重复小动作把后面补满，片头片尾全是无效空档。这是单人口播最典型的坑。
- ❌ **prompt 塞元数据 / 总时长失控** —— 把"首帧 / 时长 / 情绪标签"写进 prompt，复制即用带垃圾；或干脆不估时，成片严重超长。

这个 skill 把单人口播压成**一条管道**，核心是**切段卡时长 + 逐段干净 prompt + 防发呆**：

```
Step 1  口播稿切段      按语义切 4–15s（偏 10–15）+ 估时（数字按读法）+ 全片卡进目标时长
Step 2  角色机位锁定    一张单人固定机位首帧 host_solo.png（全集复用）
Step 3  逐段说话 prompt  每段一张卡：纯 prompt + 情绪外化 + 连贯讲话不发呆
Step 4  时长 / 防发呆校验 每段时长=台词实长、总时长≤目标、镜头数不过碎
Step 5  (可选) 数字段配图 念数字 / 数据的段配张图表 B-roll，比一直拍人耐看
```

最终交付物 = **一串逐段 prompt 卡**：每段的说话 prompt 可直接复制进即梦 / 可灵 / Vidu 抽卡。skill 本身不调任何 API。

## 核心心法（贯穿全流程）

1. **时长 = 台词实长，连贯讲话防发呆** —— 单人口播头号坑。生成时长设成该段台词念完的实际长度（**别往长里放**），prompt 里写"全程连贯讲话、话音不断、不发呆不留空档"。时长比台词长 = 模型用发呆补帧。
2. **镜头别切碎，往 10–15s 凑** —— 按语义把相邻短句合并成偏 10–15s 的段，镜头数越少、拼接缝越少、画面越不碎。
3. **卡死总时长** —— 按**念出来的字数**估时（数字按读法：4900=四千九百算 4 字、20%=百分之二十算 5 字），全片求和必须卡进目标（如 ≤90s）。
4. **一人一镜位，全集一张首帧** —— 单人固定机位，复用同一张 `host_solo.png` 当首帧，角色 / 机位 / 背景 / 光照全程一致。
5. **音画一体必开音频** —— 说话片段口型 + 声音由工具一次生成，必须开原生音频抽卡（关了口型对不上声音整段废）。
6. **prompt 正文干净、可直接复制** —— 首帧 / 音频 / 时长 / 情绪标签是元数据，写代码块外；代码块内只放能一键粘贴出片的纯 prompt。

## Step 1 — 口播稿切段

### 关键动作

- 在**句号 / 问号 / 感叹号 / 破折号 / 语气转折**处落刀；相邻短句**合并**到偏 **10–15s** 一段，整体每段 4–15s，尽量不留 <4s 的碎句。
- **估时**：按明快口播 ~4.5–5 字/秒（含强调停顿取慢端）；**数字 / 符号按读法折算字数**（4900→四千九百=4、20%→百分之二十=5、ETF→E-T-F=3），标点不计。
- **卡总时长**：各段估时**求和**，必须 ≤ 用户给的目标（如 90s）；超了就提示砍内容或按更快语速重估，别硬把台词塞进更短时长。
- 每段标**情绪**（吐槽 / 质问 / 冷静 / 转折 / 语重心长…），驱动后面 prompt 的情绪外化。
- 台词**逐字保留**，只切不改（除非用户明确要润色）。

详细切段法、数字读法表、防发呆原理见 `references/segment-and-pacing.md`。

### 雷区

- ❌ **一句一段** → 镜头碎；先合并到 10–15s（总时长 ÷ ~12s ≈ 段数，90s ≈ 7–8 段而非 15 段）。
- ❌ **不按数字读法估时** → 总时长算少了、成片超长。
- ❌ **改写用户台词** → 口播稿是用户的，默认一字不动。

### 产出物

分段表：`段号 ｜ 片段ID ｜ 台词(逐字) ｜ 情绪 ｜ 估时s`，附**总时长**与"≤目标"结论。

---

## Step 2 — 角色与机位锁定

### 关键动作

- 定 host 角色档案（脸 / 服装 / 发型 / 神情的稳定特征词）。
- 出**一张**单人固定机位首帧 `host_solo.png`：中近景、eye-level 平视、面向镜头、简洁背景、背景静止。全集所有段都用它当首帧。
- 出图配方见 `references/prompt-recipes-solo.md`。

### 雷区

- ❌ **每段换背景 / 光照 / 机位** → 段间一跳。全程复用同一张首帧、同一套锁机位词。

### 产出物

host 角色档案 + `host_solo.png` 首帧（及其出图 prompt）。

---

## Step 3 — 逐段说话 prompt

### 关键动作

每段产出**一张卡**（格式见 `references/prompt-recipes-solo.md`）：

- **段头**（代码块外）：`段号 ｜ 片段ID ｜ ~时长 ｜ 情绪`
- **台词**（逐字）
- **描述行**（代码块外）：`首帧 host_solo.png ｜ 开原生音频 ｜ ~Xs（按台词实际长度设时长）`
- **代码块**（纯 prompt）：host 形象一句 + 中近景固定机位 fixed 禁运镜 + `对着镜头连贯说出这句台词："…"` + **情绪外化成身体细节**（不写"严肃"二字，写"眉头微蹙、语速放慢"）+ **`全程连贯讲话、话音不断、不发呆不留空档`** + 口型声音精确同步、机位不动、背景静止。

### 雷区

- ❌ **把首帧 / 时长 / 情绪裸词写进代码块** → 复制即用带垃圾、污染生成。
- ❌ **只写情绪词不外化** → 表演空泛。把情绪绑到具体台词位置（`说"…"时<动作>`）。
- ❌ **漏"连贯讲话不发呆"** → 片头片尾发呆。

### 产出物

N 段说话 prompt 卡，代码块可直接复制抽卡。

---

## Step 4 — 时长与防发呆校验

### 关键动作

- 逐段：**生成时长设成台词实际长度**（如 8.5s 的段就设 ~8–9s，**别设 15s**）。
- 全片：各段时长**求和 ≤ 目标**；镜头数别过碎（偏 10–15s/段）。
- 出片后若某段片尾仍发呆：把该段时长再调短 0.5–1s 重抽，或在台词尾留个收尾动作（点头 / 收手）让结尾"有交代"。

### 雷区

- ❌ **全部按 15s 出** → 短段必发呆。
- ❌ **只看单段不看总长** → 成片超目标。

### 产出物

校验结论：总时长、镜头数、每段时长是否贴台词。

---

## Step 5 —（可选）数字 / 数据段配图 B-roll

念数字、列价格 / 数据的段，盖一张**图表 / 要点字幕卡**（B-roll）比一直拍人耐看，且口播原声保留（盖画面不盖声音）。配图 prompt 思路见 `references/prompt-recipes-solo.md`。

````

## 🎙 逐段说话 prompt 卡
### 段 01 ｜ host_say_01 ｜ ~13s ｜ 吐槽
台词："…"

``` <纯 prompt：host 形象 + 固定机位禁运镜 + 逐字台词 + 情绪外化 + 连贯讲话不发呆 + 同步约束> ```

````

## ⚠️ 需本机 / 官方复核清单（资料随版本漂移）

- ⚠️ 各工具"原生音频 / 声画同出"开关名与最低版本——以平台当前 UI 为准；说话片段必须用带原生音频的版本（可灵 3.0 / 即梦 Seedance 2.0 / Vidu Q3，⚠️ Vidu Q2 不带）。
- ⚠️ 单段时长上限（可灵约 15s、Vidu Q3 约 16s；即梦 Seedance 2.0 支持更长单段，上限随版本）、默认时长、分辨率档——以官方为准；超过所用工具单段上限的段就拆，或换支持更长单段的工具。
- ⚠️ 字/秒语速是经验值（4.5–5），按实际配音节奏校；估时只为卡总长，最终以真实时长为准。
- ⚠️ 没有任何模型保证零发呆——靠"时长贴台词 + 连贯讲话 + 复抽"压到最低，最终肉眼验收。

---

## 一句话总结

> 单人口播：按语义切成偏 10–15s 的段、按数字读法估时卡死总时长，全集复用一张固定机位首帧，逐段出**连贯讲话不发呆**的干净说话 prompt——生成时长设成台词实长，就不会"傻坐着"。
