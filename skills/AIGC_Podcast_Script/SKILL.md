# 双人 AI 对话播客（对话稿 → 三视图 → 说话片段 → 剪辑决策表）

> 人工写 A/B 对话稿 → 三视图锁机位 → 逐人生成音画一体说话片段 → **从对话稿直接算出剪辑决策表** → 剪映照表拼装

## 这个 skill 解决什么问题

用户说"我要做个两人 AI 对谈的视频/播客"。对话稿能自己写、说话片段能用即梦/可灵/Vidu 逐人生成（口型和声音生成工具自带），可一到最后剪辑就崩——**手动把一堆说话片段拼起来 + 在剪辑台上凭感觉插反应镜头，是整条管道最耗时、最折磨的一步**。

常见三种崩法：

- ❌ **最后在剪辑台拍脑袋插反应镜头** —— 最耗时、全凭感觉、做完才发现节奏全错，回炉重来。
- ❌ **把反应镜头当替换主轨用** —— 说话片段是音画一体的，一替换就连声音一起丢掉，口型和声音串台穿帮。
- ❌ **说话片段抽卡时关音频** —— 口型与声音对不上、整段作废（这点和单人分镜"抽卡省钱关音频"的直觉正好相反）。

这个 skill 把痛点压成**一条七步管道**，核心是**把剪辑决策前置到写稿之后**：

```
Step 1  对话稿结构化     A/B 逐句切分 + 情绪标签 + 时长预算
Step 2  角色与机位锁定   三视图首帧（双人/A单/B单）锁脸 + 锁机位
Step 3  镜头分配         每句台词 → 谁的固定机位说话片段（主轨骨架）
Step 4  反应镜头编排     扫触发规则 → cutaway 候选（切给谁/盖多久）
Step 5  反应素材库清单   去重后的反应片段需求（标准化命名）
Step 6  生成 prompt      逐段成对卡:说话片段(必开音频)+反应片段(无台词)
Step 7  剪辑决策表/EDL   逐行 cutaway：主轨第几秒起/盖多久/转场
                        (可选) pyJianYingDraft 一键生成剪映草稿
```

最终交付物是 **一张精确到秒的剪辑决策表（cut plan / EDL）+ 一份反应素材库需求清单 + 每个片段的生成 prompt**，让最后剪辑退化成"照表拼装"。生成视频（即梦/可灵/Vidu）与剪映拼装是用户在外部手动操作，skill 本身不调用任何 API——产出方案，用户照做并把真实文件名/时长回填。

## 核心心法（贯穿全流程）

1. **剪辑前置，照表拼装** —— 不在剪辑台凭感觉插镜头；从对话稿直接算出 cut plan，最后剪辑退化成机械执行。这条不成立，整个 skill 就没意义。
2. **说话片段音画一体，是地基不是变量** —— 口型与声音由生成工具一次出，必须**开音频抽卡**；它进主轨、保留原声、时长 = 素材真实时长，不能改不能猜。
3. **反应镜头 = 上层静音轨 cutaway，盖画面不盖声音** —— 反应片段绝不替换主轨，而是**静音后放覆盖轨（画中画/覆盖轨）**，只在某个时间窗盖住画面、主轨声音照常播（视觉上是 L-cut 效果）。这是全 skill 最关键的技术机制。
4. **角色一致性靠三视图首帧** —— 全集只锁一次三视图（双人/A单/B单固定机位首帧），所有说话片段和反应片段都复用它做首帧基础，机位天然锁定、人脸天然一致。
5. **每片段有验收标准 + 时长预算** —— 不写验收 = 抽卡全凭感觉烧额度；说话片段按台词切 4–15s，反应片段 0.5–3s，时长进决策表才能照表拼。
6. **端着，不做应声虫** —— 反应过载（几乎每句都切）、情绪错位（严肃配笑）、节奏失衡时主动叫停。知道什么时候"够了"，比知道怎么多加一个反应镜头更重要。

## Step 1 — 对话稿结构化

### 关键动作

把人工写好的对话稿**逐句切成"行"**，每行落齐六个字段。若用户只给一整段稿，先帮他切句、回填情绪标签，让他确认后再往下走。

| 字段 | 说明 |
|------|------|
| 行号 | `L01 / L02 …`，全流程引用锚点 |
| 说话人 | A / B |
| 台词 | 这一行被唱/说出来的字 |
| 情绪标签 | 严肃 / 玩笑 / 惊讶 / 质疑 / 中性 …（驱动后面反应语义） |
| 问句? | 是 / 否（驱动触发规则 T1） |
| 预估时长 | 按字数 / 语速估，落在 **4–15s**；超 15s 必拆 |

### 雷区

- ❌ **不打情绪标签** —— 后面反应语义无从对齐，反应镜头只能瞎选。
- ❌ **单句估时 > 15s 不拆** —— 超出即梦/可灵/Vidu 单段生成上限，必须拆成两段说话片段。
- ❌ **漏标问句** —— 触发规则 T1（问句→切被问方）直接失效。

### 产出物

一张**对话行表**：`行号 ｜ 说话人 ｜ 台词 ｜ 情绪标签 ｜ 问句? ｜ 预估时长`。

---

## Step 2 — 角色与机位锁定（三视图首帧）

### 关键动作

全集只锁一次。两件事：**锁角色档案** + **出三张固定机位首帧**。

1. **A/B 角色档案**：脸 / 服装 / 发型 / 光照 / 背景的稳定特征词，A、B 各一份。
2. **三视图首帧**（共享同一背景 + 同一光照 + 同一角色参考图）：
   - `two_shot.png` —— 双人同框，定两人**左右位置与朝向**（这决定了越轴方向）。
   - `A_solo.png` —— A 单人固定机位。
   - `B_solo.png` —— B 单人固定机位。
3. **复用规则**：A 的说话片段和 A 的反应片段都用 `A_solo.png` 做首帧；B 同理；双人 cutaway 用 `two_shot.png`。
4. **锁机位词必带**：`固定机位 fixed / 机位不动 / 景别不变 / eye-level`。

> 三视图/锁机位词的完整出图配方 → `references/prompt-recipes-dialogue.md`。

### 雷区

- ❌ **单人首帧与双人首帧朝向不一致** —— cutaway 盖上去会越轴跳（A 在双人里看右，单人首帧却看左）。
- ❌ **不锁背景/光照** —— 反应片段盖上主轨时画面一跳，穿帮。
- ⚠️ **双人同框讲话音画一体最不稳** —— 台词尽量靠 A单/B单的正打镜头承载，`two_shot` 多用于开场 / 换话题 / 无台词互动。

### 产出物

A/B 角色档案表 + 三张三视图首帧（`two_shot.png` / `A_solo.png` / `B_solo.png`）+ 朝向约定（如"A 看右 / B 看左"）。

---

## Step 3 — 镜头分配

### 关键动作

把对话行表逐行映射成**一个主轨说话片段**，搭出主轨骨架（= 剪辑决策表的主轨部分）：

- A 的台词 → `A_solo.png` 首帧的说话片段；B 同理。
- 为每段分配片段 ID，用统一命名 `<人物>_say_<句号两位>.mp4`，例 `A_say_03.mp4`。
- 同一人连续多句可合并成一段，也可拆成多段，逐段记录时长。

### 雷区

- ❌ **同一人两段说话片段首尾相接** —— 这是跳切风险点（同机位两段一拼会"闪"一下），**标记出来**，留给 Step 4 用反应镜头盖缝。
- ❌ **主轨片段 duration 不等于素材真实时长** —— 会黑帧/截断。**拿到生成素材后，读真实时长回填，不要一直用 Step 1 的估算值。**

### 产出物

**主轨片段清单**：`片段ID ｜ 说话人 ｜ 对应行号 ｜ 首帧 ｜ 预估时长 / 真实时长`。

---

## Step 4 — 反应镜头编排

### 关键动作

用 `references/reaction-shot-playbook.md` 的触发规则表，**逐行扫描对话行表**，命中即生成一个 cutaway 候选：标注触发规则号 + 切给谁（听者 / 被问方 / 双人）+ 盖多久 + 反应语义标签。然后应用**去重与密度规则**（间隔 ≥ 3s、长台词 >10s 必覆盖、反应/说话 ≈ 2:1，T8 遮跳切强制优先）。

SKILL.md 只放五条核心触发规则的精简摘要，完整 9 条 + 优先级去重逻辑见 playbook：

| 规则 | 命中条件 | 切给谁 | 典型反应 |
|------|----------|--------|----------|
| **T1** | 当前行是问句 | 被问方（听者） | 聆听 / 挑眉 |
| **T2** | 出现观点 / 数据 / 金句 | 听者 | 点头 / 前倾 / 做笔记 |
| **T5** | 单段台词 > 10s 的中段 | 听者 | 中性聆听（呼吸点） |
| **T8** | 同一人两段说话片段拼接缝 | 听者 | 任意（**强制盖缝遮跳切**） |
| **T9** | 开场 / 换话题 | 双人 `two` | 建立空间 |

### 雷区

- ❌ **反应过载** —— 几乎每句都切，观众晕、节奏垮。
- ❌ **情绪错位** —— 严肃数据配轻笑、玩笑配皱眉。反应情绪 = **说话方**台词情绪，不是听者自己加戏。
- ❌ **漏盖同人拼接缝（T8）** —— 跳切刺眼。
- ❌ **反应片段在动嘴** —— 无台词的反应片段一旦张嘴，口型穿帮（明明声音是对方的）。

### 产出物

**cutaway 候选列表**：`触发规则 ｜ 切给谁 ｜ 盖多久 ｜ 反应语义标签 ｜ 对应主轨行号`。

---

## Step 5 — 反应素材库需求清单

### 关键动作

把 Step 4 的 cutaway 候选按「人物 × 反应类型」**去重归并**成一张素材库清单——同一类反应（如"B 点头"）出现多次只需备 1–2 条，不必每个 cutaway 单独抽一条。每类按需多备 1–2 条防止重复露馅；一条 idle/聆听片段可在剪映里**循环 / 倒放**铺满多个时间窗，更省额度。

标准化命名（决策表"反应片段ID"列、素材库清单、生成脚本三处必须机械对齐）：

```
<人物>_react_<反应token>_<序号两位>.mp4
```
- `<人物>` = `A` / `B`（= 听者，谁出现在反应镜头里），双人位用 `two`
- `react` 固定字面，区别于说话片段
- `<反应token>` = 反应词表的英文 token（`nod` / `frown` / `chuckle` / `listen` …）
- `<序号>` = 同类多备时区分，`01` / `02`

例：`B_react_nod_01.mp4`、`A_react_frown_01.mp4`、`B_react_chuckle_02.mp4`、`two_react_listen_01.mp4`。

反应类型词表精简版（完整表 + 触发语境 + "反应情绪=说话方情绪"铁律见 playbook）：

| token | 中文 | 语义 | 建议时长 |
|-------|------|------|----------|
| `listen` | 中性聆听 | 我在听 | 1.5–2.5s |
| `nod` | 点头认同 | 同意 | 0.8–1.5s |
| `brow` | 挑眉 | 哦？有意思 | 0.6–1.2s |
| `chuckle` | 轻笑 | 缓和共鸣 | 0.8–1.5s |
| `frown` | 皱眉思考 | 我在权衡 | 1.0–1.8s |
| `shake` | 摇头 | 我不这么看 | 0.8–1.5s |

### 雷区

- ❌ **每个 cutaway 都单独抽一条** —— 同类反应重复抽，白白浪费额度。
- ❌ **命名与决策表引用列对不上** —— 拼装时（或脚本里）找不到素材。

### 产出物

**反应素材库清单**：`反应ID ｜ 人物(A/B/two) ｜ 反应类型 ｜ 时长 ｜ 机位(单人固定) ｜ 数量`。

## Step 6 — 生成 prompt

### 关键动作

为主轨说话片段、反应素材库每条分别产出 prompt。完整配方 / 三家工具差异 / 抽卡省额度技巧见 `references/prompt-recipes-dialogue.md`。

**交付形态 = 逐段成对 prompt 卡**（前端主交付物）：把每个 4–15s 段做成一张卡——① 说话片段 prompt + ② 该段配套反应片段 prompt 绑在一起，反应画面在生成环节随这段一起出、不留到剪辑；约 1/3 段为纯呼吸段只留 ①。**代码块里只放纯 prompt、能一键复制**，首帧 / 音频开关 / 时长 / 素材ID / 用途一律写在代码块外的描述行。卡片格式与范例见 `references/prompt-recipes-dialogue.md` 的「逐段成对 prompt 卡」。

**说话片段 prompt（主轨）：**
- 用对应单人三视图首帧（`A_solo.png` / `B_solo.png`）做首帧。
- **台词写进 prompt** + **必开音频**（音画一体、口型由工具生成）。
- 固定机位禁运镜 + 情绪外化（按 Step 1 情绪标签）。

**反应片段 prompt（覆盖轨）：**
- 用**同一张**单人首帧（机位/光照/景别与说话片段一致）。
- **无台词**（绝不张嘴）+ 微动作（按反应词表 token）。
- 首尾一致、loop 友好；关音频或事后在剪映静音。

### 雷区

- ❌ **说话片段关音频抽** —— 和单人分镜"抽卡关音频省钱"正好相反，这里**必须开**，否则口型声音对不上整段废。
- ❌ **反应片段机位/光照/景别与说话片段不一致** —— 盖上去画面跳。
- ❌ **把首帧/音频/时长/用途写进 prompt 正文** —— prompt 要干净到能一键复制；这些元数据放代码块外的描述行。
- ⚠️ **没有任何模型零穿帮** —— 靠"短 + 小动作 + 同机位 + 多抽挑"把概率压低，最终肉眼验收。

### 产出物

每个主轨片段一段说话 prompt + 每条反应素材一段反应 prompt + 每片段验收 checklist。

---

## Step 7 — 剪辑决策表 / EDL（+ 可选自动生成剪映草稿）

### 关键动作

把**主轨片段清单 + cutaway 候选**合成最终剪辑决策表。**每一行 = 上层覆盖轨的一个 cutaway**，精确到秒。设计契约：列与 `pyJianYingDraft` 的 API 参数一一对应，使"表 → 手动剪辑"和"表 → 脚本生成草稿"都是机械映射。

核心字段（完整字段释义 + 取值约束 + 与主轨清单的关系图见 `references/cut-plan-and-capcut.md`）：

| 列 | 含义 | 约束 |
|----|------|------|
| 序号 | cutaway 行号 | 1,2,3… |
| 对应台词 | 盖在哪句台词上 | 引用 Step 1 行号 |
| 主轨片段ID | 被盖的说话片段 | 如 `A_say_03` |
| 主轨入点(s) | 该片段在时间线起点 | 前序片段时长累加 |
| 主轨时长(s) | = **素材真实时长** | 必须等于真实时长 |
| 反应片段ID | 用哪条反应素材 | 如 `B_react_nod_01` |
| cutaway主轨起点(s) | 覆盖片段在主轨第几秒开始 | ≥ 主轨入点，落句末/换气 |
| cutaway时长(s) | 盖多久 | 0.5–3.0；遮跳切 ≥ 接缝两侧各 0.3s |
| 转场 | 进出转场 | 默认 **硬切**（不加 transition） |
| 静音 | 覆盖片段是否静音 | **恒为「是」（硬性）** |
| 触发规则 | 命中的规则号 | T1–T9 |

> 上表是"表 → 手拼 / 表 → 脚本"机械映射所需的**核心列**；`是否叠加反应 / 情绪标签 / 备注` 属扩展列，完整 14 列 schema（含纯说话片段「叠加=否」行的写法）见 `references/cut-plan-and-capcut.md`。`cutaway主轨起点(s)` 指该 cutaway 在**时间线上的绝对秒**，不是片段内相对秒。

示例（节选）：

| 序号 | 对应台词 | 主轨片段ID | 主轨入点 | 主轨时长 | 反应片段ID | cutaway主轨起点 | cutaway时长 | 转场 | 静音 | 规则 |
|------|----------|------------|----------|----------|------------|-------------|-------------|------|------|------|
| 1 | L03 A「关键在于复利」 | A_say_03 | 6.0 | 7.0 | B_react_nod_01 | 9.5 | 1.5 | 硬切 | 是 | T2 |

### （加速档）pyJianYingDraft 自动生成剪映草稿（先本机验证）

**主推但有前提。** 决策表每行 → 一段代码，一键生成剪映草稿，省掉手拼。全新生成草稿天然绕开剪映模板加密，但带版本/平台前提，**主流程不依赖它，请先在本机验证再用**。要点（完整脚本骨架 + 坑见 `references/cut-plan-and-capcut.md`）：

- **库与安装**：国内剪映 → `pyJianYingDraft`（`pip install pyJianYingDraft`，最成熟首选）；国际 CapCut → `pyCapCut`（`pip install pycapcut`，**实验性、能力略弱、迁移中**）。
- **建两条 video track**：`main`（主轨）+ `cutaway`（覆盖轨），用 `relative_index` 分层。
- **主轨**：`add_segment(seg, "main")` 保留原声；`duration` 用素材真实时长。
- **覆盖轨**：`VideoSegment(..., volume=0.0)` 静音 + `trange(起点, 时长)` 精确落点；硬切 = 不调 `add_transition`；最后 `script.save()`。
- **草稿默认路径**：剪映里看「草稿参数 → 保存位置」最稳（路径/主数据文件名随版本平台变）。

### 雷区

- ❌ **忘记静音覆盖片段** —— 反应片段的"环境音/呼吸声"和主轨说话声串台。
- ❌ **主轨 duration 填估算值而非真实时长** —— 黑帧/截断。
- ❌ **跨机器素材绝对路径变化** —— 换机/移动文件夹后需在剪映「重新链接素材」；素材+草稿放固定稳定目录。
- ⚠️ **`relative_index` 越大越靠上层** —— 库文档：值越大越接近前景，所以覆盖轨取比主轨大的值（脚本里 `main=1 / cutaway=2` 方向正确）；生成后在剪映里扫一眼确认即可。

## 常见翻车场景与应对

| 翻车场景 | 现象 | 应对 |
|---------|------|------|
| **替换主轨丢声音** | 反应镜头一盖，对应那句话没声了 | 反应片段绝不替换主轨，**静音放上层覆盖轨**做 cutaway，主轨原声保留 |
| **忘静音串台** | 主轨说话声 + 反应片段环境音双声打架 | 覆盖片段恒 `volume=0` / 剪映里手动静音（决策表"静音"列永远是） |
| **说话片段关音频** | 口型与声音对不上、整段废 | 说话片段**必开音频**抽卡（与单人分镜相反） |
| **反应过载** | 几乎每句都切、观众晕 | 应用密度规则：间隔 ≥ 3s、反应/说话 ≈ 2:1 |
| **情绪错位** | 严肃数据配轻笑 | 反应情绪 = 说话方台词情绪，按情绪标签选 token |
| **机位不一致跳** | 反应片段盖上去画面一闪 | 反应片段与说话片段同首帧/同机位/同光照/同景别 |
| **同人拼接缝跳切** | 同一人两段一拼"闪"一下 | T8 强制用反应盖缝，cutaway 时长 ≥ 接缝两侧各 0.3s |
| **跨机器素材失链** | 换机打开草稿素材变灰 | 素材用绝对路径、放固定目录；换机后剪映「重新链接素材」 |
| **relative_index 层级反了** | 反应片段被主轨盖住没显示 | ⚠️ 生成后在剪映看哪条轨在上层，调一次 relative_index |

## ⚠️ 需本机验证清单（资料随版本/平台漂移，用前自查）

- ⚠️ `VideoSegment(volume=0.0)` 在你的剪映版本对视频轨的实际静音效果——以本机实际播放为准。
- ⚠️ 草稿默认路径与主数据文件名随版本/平台变化（macOS 历史上有 `draft_info.json`）——以剪映「草稿参数 → 保存位置」为准。
- ⚠️ **自动导出依赖 `uiautomation`、仅 Windows 支持（与剪映版本无关）；库 README 明确草稿"仍需在 Windows 版剪映下导出"** —— macOS/Linux 能否在本机剪映 GUI 直接导出该草稿未经验证，**必须本机自测，别默认"打开剪映点导出即可"**。新草稿个别版本需重启剪映才出现在首页列表。
- ⚠️ 说话片段必须用**带原生音频**的工具版本（可灵 3.0 / 即梦 Seedance 2.0 / Vidu Q3；旧版可能不带）——以平台当前 UI 为准。
- ⚠️ 单段时长上限、分辨率档、seed 复现、价格/积分各平台不一——用前现查官方。
- ⚠️ 触发规则的时长档、密度档、2:1 比、间隔 ≥ 3s 均为经验起点——按内容节奏与本人审美微调。
- ⚠️ 没有任何生成模型零穿帮——反应镜头靠"短 + 小动作 + 多抽挑 + 同机位"压低概率，最终肉眼验收。

## 一句话总结

> 把剪辑决策前置到写稿之后：从 A/B 对话稿直接算出一张精确到秒的 cut plan + 反应素材库清单，**说话片段音画一体进主轨、静音反应片段进上层覆盖轨做"盖画面不盖声音"的 cutaway**，让最后剪映拼接退化成照表拼装。
