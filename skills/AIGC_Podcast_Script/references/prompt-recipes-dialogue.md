# 即梦 / 可灵 / Vidu 图生视频 prompt 配方（双人对话播客）

SKILL.md **Step 2**（出三视图首帧）和 **Step 6**（写说话片段 / 反应片段 prompt）调用本文件。本文件只讲**生成侧**：三视图怎么出、说话片段怎么写、反应片段怎么写、三家工具怎么选、一致性怎么保、怎么抽卡省额度、每镜怎么验收。

> 不写剪辑节奏 / 触发规则（→ `reaction-shot-playbook.md`），不写决策表字段 / 剪映脚本（→ `cut-plan-and-capcut.md`）。

---

## ⚠️ 一条铁律先记住：说话片段**必须开音频**抽卡

和单人分镜「抽卡关音频省钱」的直觉**正好相反**。说话片段是**音画一体**的——口型和声音由生成工具一次出，关了音频抽出来的口型对不上后配的声音，整段作废。所以：

- **说话片段** → 开原生音频（`generate_audio` / 声画同出模式），台词写进 prompt，必须用**带原生音频**的工具版本：**可灵 3.0 / 即梦 Seedance 2.0 / Vidu Q3**。⚠️ Vidu Q2 不带原生音频，必须升 Q3。
- **反应片段** → 无台词、可关音频抽（关不掉就事后在剪映静音）。它进上层覆盖轨、恒静音，原生音频用不上。

⚠️ 各工具「原生音频 / 声画同出」开关名与最低版本随平台 UI 变，用前现查官方。

---

## ⚠️ 第二条铁律：prompt 正文必须干净、可直接复制

代码块里**只放纯 prompt**——能一键复制粘贴进即梦 / 可灵 / Vidu 的文本框就出片。**首帧文件、音频开关、duration、素材ID、用途/触发**这些是元数据，一律写在代码块**外面**的描述行里，绝不混进 prompt 正文。

- ❌ prompt 里出现 `首帧：A_solo.png` / `[开原生音频，duration=13s]` / `（用途：盖爆点）` —— 复制即用会带垃圾、污染生成。
- ✅ prompt 只描述画面：人物 + 景别机位 + 台词（说话片段）/ 微动作（反应片段）+ 情绪 + 约束。元数据放卡的描述行：`首帧 A_solo.png ｜ 开原生音频 ｜ ~13s`。

## (1) 三视图首帧出图配方（Step 2 — 锁机位锁人）

**「三视图」不是角色正背侧三视图，而是本工作流定义的「三个固定机位首帧」：双人同框、A 单人、B 单人。** 它们是全集所有片段的出图 / 首帧底座——说话片段用对应单人首帧，反应片段用同一张单人首帧，开场 / 换话题 / 无台词互动用双人首帧。机位由首帧天然锁定。

### 出图顺序与共享变量

```
1. 先出 two_shot.png（双人同框）→ 定下两人左右位置、朝向、背景、光照、机位高度
2. 再出 A_solo.png / B_solo.png，要求与 two_shot 共享：
   - 同一角色参考图（脸/服装/发型）
   - 同一背景
   - 同一光照（方向 + 色温）
   - 同一机位高度（eye-level 平视）
   - 朝向 = 双人镜头里各自的朝向（A 在双人里看右 → A_solo 也看右）
```

⚠️ **单人首帧朝向必须和双人首帧一致**，否则反应片段盖上主轨会越轴跳（A 在双人里看右、单人却看左 = 空间关系崩）。

### 双人首帧 `two_shot.png` 配方

```
中近景双人对坐播客场景，左侧人物 A：[A 稳定特征词]，朝向右侧；
右侧人物 B：[B 稳定特征词]，朝向左侧；两人隔桌相对、自然交谈姿态。
背景：[固定背景，如 暖色书架 / 录音棚吸音棉墙]，背景静止；
光照：[左前方暖光 4500K，柔光]；
固定机位 fixed camera，eye-level 平视，机位不动，景别不变。
Negative：运镜、推拉摇移、变形脸、多余的人、文字水印。
```

### 单人首帧 `A_solo.png` / `B_solo.png` 配方

```
首帧参考：two_shot.png（或角色参考图）。
中近景单人播客镜头，人物 A：[与 two_shot 完全一致的 A 特征词]，
朝向右侧（= 双人镜头里的朝向），自然端坐、面向画外的交谈对象；
背景与光照同 two_shot（[同一句背景描述] + [同一句光照描述]）；
固定机位 fixed camera，eye-level，medium close-up 中近景，机位不动。
Negative：运镜、朝向相反、换背景、换光照、变形脸、文字水印。
```

> B 单人把 `A` 换成 `B`、朝向改「左侧」即可。

### 锁机位词（三张首帧 + 后续所有视频 prompt 都要带）

`固定机位 fixed camera / 机位不动 / 景别不变 / eye-level 平视 / medium close-up 中近景 / 背景不变 background unchanged`

⚠️ **双人同框讲话的音画一体目前最不稳**（多角色口型 / 声音容易串台）。台词尽量靠 A 单 / B 单的正打镜头承载；`two_shot` 多用作开场 / 换话题 / 无台词互动。可灵 3.0 对双人同框讲话相对最稳（可分别控制每人说话风格），即梦 / Vidu 双人同框建议留给无台词镜头。

## (2) 说话片段配方（Step 6 主轨 — 音画一体、必开音频）

**入口：图生视频「首帧 + 文 + 音」。首帧用对应单人三视图（A 用 `A_solo.png`，B 用 `B_solo.png`），台词写进 prompt（或挂音频素材），让模型原生出口型 + 声音。**

### 说话片段 prompt 黄金结构

> 下面是**结构拆解**（带占位符，非成品）。成品卡里 `首帧 / 音频 / duration` 写在代码块外的描述行，代码块只留「文字 prompt」那几行——见文末「逐段成对 prompt 卡」与上面第二条铁律。

```
首帧：A_solo.png（单人固定机位，平台界面手动上传）

文字 prompt：
[人物设定：与三视图一致的稳定特征词，一句带过即可，首帧已表达]
+ [景别 + 固定机位 fixed，禁止任何运镜]
+ [台词（音画一体）：自然说出这句台词："……"]
+ [情绪外化：把 Step 1 情绪标签翻成身体细节，不写"严肃"写"眉头微蹙、语速放慢"]
+ [音频设计：对话清晰、轻微室内底噪、无背景音乐]
+ [约束：口型与声音精确同步、表情自然、机位全程不动]

参数：开原生音频 / 声画同出，duration = 该句预估时长（4–15s）
```

### 参数取向

| 维度 | 取向 |
|------|------|
| **音频** | **必开**原生音频（与单人分镜相反）。台词写进 prompt 或挂音频素材。 |
| **时长** | 按 Step 1 单句预估切，4–15s 一段（可灵 3–15s、即梦 4–15s、Vidu Q3 最长 16s）。超 15s 在 Step 1 就该拆。 |
| **分辨率** | 抽卡用低分辨率预览验口型 / 构图，定稿再出高清（⚠️ Vidu Q3 据单一来源支持 320P 预览，以官方为准）。 |
| **运镜** | 留空或显式 `fixed`，**绝不**写 dolly / pan / 推拉摇移。固定机位播客就该一动不动。 |
| **情绪** | 外化成身体细节（「肩膀微颤、眼眶泛红、嘴角上扬」）比写情绪词更准，口型 / 表情更贴台词。 |
| **seed** | 满意的镜头记下 seed 复现（⚠️ 各平台 seed 支持程度不一）。 |

### 说话片段 prompt 示例

首帧 `A_solo.png` ｜ 开原生音频 ｜ ~6s（**这些写在卡的描述行，不进 prompt 正文**），代码块即成品可直接复制：

```
藏青针织衫、细框眼镜的男主播，中近景，固定机位 fixed，朝向右侧，对着画外的交谈对象自然说出这句台词："这个观点我其实有点保留……"。语气平和、语速略慢，说话时偶尔轻点头、眉毛微挑；环境为安静室内，对话清晰、轻微底噪、无背景音乐；口型与声音精确同步，表情自然，机位全程不动。
```

## (3) 反应片段配方（Step 6 覆盖轨 — 无台词、loop 友好、小动作、同机位同人）

**这是把「剪辑前置」落地的素材库环节。每个反应片段必须：无台词、同机位、同人、动作幅度小、首尾一致 loop 友好。** 它进上层覆盖轨、恒静音，只盖画面不盖声音。

### 通用要求（所有反应片段共用）

- **首帧 = 对应说话片段的同一张单人首帧**（`A_solo.png` / `B_solo.png`，双人位用 `two_shot.png`）。机位 / 景别 / 光照 / 朝向必须和说话片段**完全一致**，否则盖上去跳。
- **无台词、嘴部静止或自然微动**（绝不张嘴说话——明明声音是对方的，张嘴 = 口型穿帮）。
- **动作幅度小、缓慢自然**：幅度越小越不容易穿帮、越容易首尾衔接。
- **首尾状态一致 / 原地不动 / 位置固定**：剪映里可循环或正放 + 倒放铺更长的窗口（一鱼多吃，省额度）。
- **时长出 4–6s 即可**（覆盖轨实际只取 0.5–3s，多出的余量给剪映挑 loop 点）。
- **关音频抽**（关不掉就事后静音）。

### 反应片段 prompt 通用骨架

首帧 `A_solo.png`（与说话片段同一张）｜ 静音 ｜ 4–6s（**写在描述行，不进 prompt 正文**），代码块即成品：

```
固定机位 fixed，无台词，人物保持坐姿不动、嘴部静止；[微动作：按反应类型选一种，见下表]；动作幅度小、缓慢自然；首帧与尾帧姿态尽量一致以便循环 loop；环境与光照同说话片段；不说话、不张嘴、机位不动、位置固定。
```

### 按反应类型的微动作模板（token 对齐 SKILL.md / 词表）

| token | 中文 | 微动作 prompt 片段（填进上面骨架的 `[微动作]`） |
|-------|------|-----------------------------------------------|
| `listen` | 中性聆听 | 自然眨眼、呼吸起伏、目光看向画外说话者，神情专注 |
| `nod` | 点头认同 | 轻轻点头两下，目光看向画外，嘴角微抿表示认同 |
| `lean` | 身体前倾 | 上身极缓慢向前倾一点点、目光专注，表示「我很感兴趣」 |
| `brow` | 挑眉 | 眉毛轻轻上挑一下、眼睛微睁，表示「哦？有意思」 |
| `surprise` | 惊讶 | 眉毛上挑、眼睛微睁、嘴唇微张（不出声），表示意外 |
| `chuckle` | 轻笑 | 嘴角上扬、肩膀轻微一抖，无声轻笑 |
| `laugh` | 大笑 | 笑容展开、头微仰、肩膀起伏（无声，事后静音），情绪高点 |
| `frown` | 皱眉思考 | 微皱眉、目光下移片刻、缓慢眨眼，表示在权衡 |
| `shake` | 摇头 | 轻轻左右摇头、嘴角下抿，表示不认同 |
| `note` | 做笔记 / 低头 | 目光下移、轻微低头像在记录，神情认真 |
| `hold` | 欲言又止 | 嘴唇微动想插话又收住、目光闪烁，张力期待（**注意：嘴只能微动不能成型说话**） |
| `two` | 双人中性 | 用 `two_shot.png`，两人自然交谈姿态、目光交流、无台词，建立空间 |

### loop 友好 & 不穿帮要点（雷区）

- ❌ **反应片段在动嘴成型说话** —— 口型穿帮（声音是主轨对方的）。`hold` / `chuckle` 这类要特别盯，嘴只能微动。
- ❌ **机位 / 光照 / 景别和说话片段不一致** —— 盖上去画面一跳。必须同一张首帧出。
- ❌ **动作幅度大 / 起止姿态差很多** —— 没法 loop，剪映里硬循环会「弹」一下。
- ⚠️ **没有任何模型零穿帮** —— 反应片段靠「短 + 小动作 + 同机位 + 多抽 1–2 条挑」把概率压到最低，最终肉眼验收。
- 💡 每种反应类型按需多备 1–2 条防重复露馅；一条 `listen` / `idle` 可在剪映循环 / 倒放铺满多个时间窗，不必每个 cutaway 单独抽。

## 逐段成对 prompt 卡（Step 6 的交付形态 ★）

把对话稿按 4–15s 切好段后（Step 1），**每段产出一张「成对 prompt 卡」**：① 说话片段 prompt + ② 该段配套反应片段 prompt 绑在一起。反应画面就在生成环节随这段一起出，不留到剪辑——这是本 skill 前端的主交付物，比剪辑决策表更先用到。

### 卡片格式（代码块只放纯 prompt，元数据全在外面）

- **段头**：`段序号 ｜ 片段ID ｜ 说话人 ｜ ~时长 ｜ 情绪 ｜ 反应类型(或"无")`
- **台词**：该段说出来的字（保留口语，是否润色由用户定）
- **① 说话片段**：描述行写 `首帧 X_solo.png ｜ 开原生音频 ｜ ~Xs`，下面代码块放纯 prompt
- **② 反应片段**：描述行写 `(用途：…) ｜ 首帧 Y ｜ 静音 ｜ 4–6s ｜ 素材 <reaction_id>`，下面代码块放纯 prompt
- **呼吸段**（约 1/3 的段）只留 ①，并标「纯主轨呼吸段」

### 范例（一张完整卡）

段 05 ｜ `A_say_05` ｜ 林深(A) ｜ ~13s ｜ 严肃·强调 ｜ 反应：晓雯 惊讶
台词："但是！SpaceX 今年净利润大概四五十亿美元，对应 1.77 万亿估值，市盈率大概三四百倍。"

**① 说话片段**　首帧 `A_solo.png` ｜ 开原生音频 ｜ ~13s
```
林深朝右，中近景固定机位，禁止任何运镜。严肃强调、语速放慢地说出台词："但是！SpaceX今年净利润大概四五十亿美元，对应1.77万亿的估值，市盈率大概在三四百倍。"。眉头微蹙，说到"三四百倍"时身体微前倾、右手轻压桌面、目光笃定。口型与声音精确同步，机位全程不动，背景静止。
```
**② 反应片段**　(用途：盖"三四百倍"爆点) ｜ 首帧 `B_solo.png` ｜ 静音 ｜ 4–6s ｜ 素材 `B_react_surprise_01`
```
晓雯朝左，中近景固定机位。无台词：眉毛上扬、眼睛微微睁大、上半身轻轻一顿，惊讶但克制，嘴部基本闭合不出声。动作收敛，首尾姿态一致便于循环。机位、光照、朝向与说话片段一致，背景静止。
```

### 雷区

- ❌ **元数据混进 prompt 正文** —— 首帧 / 音频 / 时长 / 素材ID / 用途一律放代码块外（见上面「第二条铁律」）。
- ❌ **每段都配反应** —— 约 1/3 段留作纯呼吸段（只出 ①），避免反应过载（密度规则见 `reaction-shot-playbook.md`）。
- 💡 反应片段标 `素材 <reaction_id>`，同类复用同一条（如多处 `B_react_nod_01`）→ 一条生成、多段复用，不必每段重抽。

## (4) 跨片段角色一致性策略

**核心原则：全集只锁一次「三视图首帧 + 角色参考图」，之后所有片段都复用它当出图 / 首帧基础。** 一致性在 2026 后已是模型原生能力（多图参考 + 跨镜头身份绑定），但以下变量仍要人工卡死，不能甩给模型自动推断。

| 维度 | 锁定方法 |
|------|---------|
| **面部** | 同一张角色参考图（人脸特写）贯穿全集；可灵用「主体参考 / subject reference」，即梦用 `@图片名` 引用，Vidu 用参考图槽位 |
| **机位** | 全集复用同一套三视图首帧；固定机位词必带 `固定机位 fixed`，禁止任何运镜 |
| **服装 / 发型** | 写进 prompt 的稳定特征词（「藏青针织衫、细框眼镜、低马尾」），逐镜复述，不靠图自动推断 |
| **光线** | 固定光源方向 + 色温词（「左前方暖光 4500K、柔光」），单人镜头与双人镜头用**同一句**光照描述 |
| **背景** | 同一背景参考图 / 同一句背景描述，播客背景静止即可 |
| **朝向** | 单人首帧朝向 = 双人镜头里各自朝向（A 看右 / B 看左），决定越轴方向，全集不变 |

### 参考图配比

最高支持多张（Vidu Q2/Q3 实测最高 7 张），但 **3–4 张最佳**：一张人脸 + 一张全身 + 一张场景（+ 一张道具）。⚠️ 实际上限与最佳张数随平台版本变，以官方为准。

### 一致性中→英词表（写进 prompt 直接用）

| 中文 | 英文 |
|------|------|
| 与参考图同一角色 | same character as reference |
| 主体参考锁定 | subject reference locked |
| 服装与发型保持一致 | consistent outfit and hairstyle |
| 跨镜头身份一致 | consistent identity across shots |
| 固定机位 | fixed camera |
| 机位不动 | static camera / camera does not move |
| 同一光照 | same lighting |
| 背景不变 | background unchanged |
| 景别不变 | same shot size |
| 平视 | eye-level |
| 中近景 | medium close-up |

## (5) 镜头语言 / 一致性中→英词表（生成 prompt 通用）

播客固定机位场景用得到的精简词表（构图全表见 `video_storyboard/references/image-prompt-recipes.md`；本表只挑对话播客高频项）。

### 镜头语言

| 中文 | 英文 |
|------|------|
| 中近景（播客默认） | medium close-up |

### 每镜验收 checklist

**说话片段验收（主轨）：**
- [ ] 口型与声音**逐字同步**，无错位、无飘音
- [ ] 面部 / 服装 / 发型与角色参考图一致，无崩脸
- [ ] 机位完全静止，无意外运镜、无背景漂移
- [ ] 声音干净（无怪异底噪 / 电音），情绪与台词匹配
- [ ] 时长覆盖该句台词，首尾无突兀截断
- [ ] 朝向正确（与三视图约定一致：A 看右 / B 看左）

**反应片段验收（覆盖轨）：**
- [ ] 无台词、嘴部不成型说话（静止或自然微动）
- [ ] 机位 / 景别 / 光照 / 朝向与对应说话片段**完全一致**（盖上去不跳）
- [ ] 动作幅度小、自然，无穿帮（手 / 眼 / 肩不漂移）
- [ ] 首尾姿态接近、可循环 loop
- [ ] 已（或可）静音，准备放上层覆盖轨
- [ ] 命名规范，与决策表「反应片段ID」列对得上（`B_react_nod_01.mp4`）

### 抽卡省额度技巧

1. **低分辨率预览先验 prompt / 构图 / 口型**，定稿再出高清（⚠️ Vidu Q3 据称 320P 预览最省，以官方为准）。
2. **分阶段抽**：先抽说话片段（必带音频）定稿 → 再批量抽反应片段（短、便宜、可关音频）。
3. **固定 seed 复现**满意结果，微调时锁 seed 只改 prompt，减少盲抽（⚠️ seed 支持程度各平台不一）。
4. **反应片段一鱼多吃**：一条 `listen` / `idle` 在剪映循环 / 倒放可铺多个 cutaway 窗口，不必每个 cutaway 单独抽。
5. **用每日免费积分**跑预览，高清出片再用付费额度。
6. **把播客镜头写得「越简单越好抽」**：固定机位 + 小动作的镜头抽卡次数应远低于行业实测复杂镜的 30–40 次（⚠️ 经验值，按实际为准）。一般固定机位说话片段 ⭐⭐⭐（口型是难点）、反应片段 ⭐⭐（小动作好出）。

## 雷区汇总（生成侧）

| 雷区 | 规避 |
|------|------|
| **说话片段关音频抽** | 必开原生音频（与单人分镜相反），否则口型对不上整段废 |
| **反应片段在张嘴说话** | 无台词、嘴部静止 / 微动，`hold`·`chuckle` 尤其盯 |
| **反应片段机位 / 光照不一致** | 同一张三视图首帧出，朝向 / 景别 / 背景全一致 |
| **双人同框讲话不稳** | 台词靠 A 单 / B 单正打承载，`two_shot` 留给无台词镜头（双人讲话用可灵 3.0 最稳） |
| **单句 > 15s 没拆** | Step 1 就该拆，超即梦 / 可灵 / Vidu 单段上限 |
| **高度写实人脸崩脸 / 审核** | 多抽几条挑，或轻度风格化 |
| **没存 seed / prompt** | 出好的镜头存 seed + 完整 prompt，下集同 IP 复用 |
| **用了不带原生音频的旧版** | 说话片段必须用可灵 3.0 / 即梦 Seedance 2.0 / Vidu Q3（⚠️ Vidu Q2 不带） |
