# 反应镜头编排 Playbook（触发规则 + 反应词表 + 节奏 + 雷区）

> SKILL.md 在 **Step 4 反应镜头编排**、**Step 5 反应素材库需求清单** 调用本文件。
> 它只回答四件事：**何时切、切给谁、切多久、配什么反应语义**。
> 不写决策表字段 schema 与剪映操作（→ `references/cut-plan-and-capcut.md`），不写怎么用 AI 生成反应片段的 prompt（→ `references/prompt-recipes-dialogue.md`）。
> 全文目标：把"凭感觉插反应镜头"变成**可从对话稿文本直接判定**的规则，让 AI 逐行扫描对话行表就能产出每个 cutaway 候选。

---

## 0. 前置技术机制：盖画面不盖声音（先理解，否则下文全错）

说话片段是**音画一体**——口型 + 声音由即梦/可灵/Vidu 一次生成、绑死在一起。所以反应镜头**绝不替换主轨**，而是叠加：

| 轨道 | 放什么 | 音频 | 作用 |
|------|--------|------|------|
| 主轨（下层） | A/B 说话片段（音画一体） | **保留原声** | 声音基准，全程不断 |
| 覆盖轨 / 画中画（上层） | 无台词反应片段 | **静音（volume=0）** | 只在某个时间窗盖住画面 |

效果 = **视觉上的插入镜头 / L-cut**：观众看见听者在反应，但听见的还是说话方的声音。本 skill 所有 cutaway 都是这一种实现，**决策表的每一行 = 上层覆盖轨的一个片段**。

> 判定红线：覆盖片段**必须先静音**。不静音 = 反应片段的环境音/呼吸声和主轨说话声双声串台、口型对不上音 = 立刻穿帮。"静音"在决策表里是恒为「是」的硬性字段，不是可选项。

## 1. 触发规则表 T1–T9（什么时机切到反应）

每条规则给出**可判定条件**——AI 能从对话行表的文本（标点、关键词）或时间轴（连续说话时长、拼接缝）直接判定。逐行扫描对话行表，命中即生成一个 cutaway 候选。反应类型用 §2 词表的英文 token。

| # | 触发场景 | 可判定条件（从文本/时间轴识别） | 切给谁 | 默认反应 token | 默认时长 | 转场 |
|---|----------|--------------------------------|--------|----------------|----------|------|
| **T1** | **问句之后** | 当前行问句?=是（台词以 `？/吗/呢/对吧/你觉得/是不是` 结尾） | **被问方**（提前露出"准备答"） | `listen` / `brow` | 0.8–1.5s | 硬切 |
| **T2** | **金句/观点/数据落地** | 出现数字，或 `我认为/关键在于/最重要的是/本质上/核心是` | **听者** | `nod` / `lean` / `note` | 1.0–2.0s | 硬切 |
| **T3** | **情绪转折** | 出现 `但是/然而/没想到/居然/反过来/结果` | **听者** | `brow` / `surprise` | 0.6–1.2s | 硬切 |
| **T4** | **被反驳 / 不同意** | 出现 `我不这么看/未必/恰恰相反/不对/不一定` | **被反驳方** | `frown` / `shake` / `hold` | 1.0–1.8s | 硬切 |
| **T5** | **长台词换气处** | 单方连续说话 **>8–10s**，取句末停顿点 | **听者**（同时盖说话方潜在跳切） | `listen` / `nod` | 1.5–2.5s | 硬切 |
| **T6** | **笑点 / 玩笑** | 出现 `哈哈/开玩笑/说真的/你猜怎么着`，或情绪标签=玩笑 | **听者** | `chuckle` / `laugh` | 0.8–1.5s | 硬切 |
| **T7** | **句末自然停顿** | 句号 + 说话方明显气口（节奏空档） | **听者** | `listen` / `nod` | 0.6–1.2s | 硬切 |
| **T8** | **遮丑：拼接断点** | 同一人两个说话片段首尾相接（同机位→**跳切**风险，Step 3 已标记的接缝） | **对方** | 任意（语义对得上即可） | ≥ 接缝两侧各 0.3s | 硬切 |
| **T9** | **话题切换 / 开场** | 段落起始、抛出新主题、首行 | **双人镜头** `two` | `two` | 1.5–3.0s | 硬切 |

### 优先级与去重（AI 必须执行，否则会反应过载）

1. **T8（遮跳切）= 强制项**——同人拼接缝必须有覆盖，否则画面会"跳"。其余 8 条是增强项，可删可合。
2. **同一时间窗多条命中** → 只保留优先级最高一条：
   `T8 > T4 > T3 > T2 > T6 > T1 > T7 > T5`
   （遮丑最刚需 → 戏剧张力 → 信息落地 → 情绪点 → 问答衔接 → 常规节奏垫场）
3. **两个 cutaway 间隔 < 3s** 视为过密 → 合并相邻的，或丢弃低优先级者（见 §4 密度）。
4. **去重后再过一遍 §2 语义对齐铁律**：选用 token 的情绪必须 = 说话方台词情绪标签，错位的就地改 token。

> 扫描产出 = cutaway 候选列表：`触发规则 ｜ 切给谁 ｜ 盖多久 ｜ 反应 token ｜ 对应主轨行号`。这张表去重归并后进 Step 5，落点精确化后进 Step 7 决策表。

## 2. 反应类型标准词表（情绪/语境 → 反应 → 语义）

这是**与 SKILL.md / 决策表 / 素材库命名共用的唯一命名源**。英文 token 用于命名 `<人物>_react_<token>_<序号>.mp4`；中文名/语义用于决策表自检；触发语境/配套规则用于反查"该选哪个 token"。SKILL.md 放精简 6 行，全表在此。

| token | 中文 | 触发语境 | 传达语义 | 建议时长 | 配套规则 |
|-------|------|----------|----------|----------|----------|
| `listen` | 中性聆听 | 长台词中段、过渡、问句后 | 我在听 | 1.5–2.5s | T1/T5/T7 |
| `nod` | 点头认同 | 观点/数据落地、附和 | 同意、对 | 0.8–1.5s | T2/T5/T7 |
| `lean` | 身体前倾 | 金句、关键信息 | 我很感兴趣 | 1.0–1.8s | T2 |
| `brow` | 挑眉 | 意外信息、轻转折、问句后 | 哦？有意思 | 0.6–1.2s | T1/T3 |
| `surprise` | 惊讶（睁眼/张口不出声） | 数据反差、爆点 | 没想到 | 0.6–1.2s | T3 |
| `chuckle` | 轻笑 | 玩笑、轻松点 | 缓和、共鸣 | 0.8–1.5s | T6 |
| `laugh` | 大笑 | 强笑点 | 情绪高点 | 1.0–2.0s | T6 |
| `frown` | 皱眉思考 | 复杂论点、质疑、权衡 | 我在权衡 | 1.0–1.8s | T2/T4 |
| `shake` | 摇头 | 不同意、否定 | 不对、我不这么看 | 0.8–1.5s | T4 |
| `note` | 做笔记/低头 | 干货密集、数据 | 值得记下来 | 1.2–2.0s | T2 |
| `hold` | 欲言又止 | 被反驳、想插话 | 张力、期待回应 | 0.8–1.5s | T4 |
| `two` | 双人中性 | 开场、换话题、段落收尾 | 建立空间/关系 | 1.5–3.0s | T9 |

### 语义对齐铁律（防情绪错位，最常翻车点）

**反应情绪 = 说话方台词的情绪，不是听者自己加戏。** 决策表每行都带"台词情绪标签"，AI 据此选 token、并自检：

| 说话方台词情绪 | ✅ 合法 token | ❌ 禁用 token |
|---------------|---------------|---------------|
| 严肃 / 数据 / 干货 | `nod` `note` `frown` `lean` `listen` | `chuckle` `laugh`（严肃配笑=出戏） |
| 玩笑 / 轻松 | `chuckle` `laugh` `brow` | `frown` `note`（玩笑配皱眉思考=尴尬） |
| 惊讶 / 爆点 / 转折 | `surprise` `brow` `lean` | `note`（没人爆点还低头记笔记） |
| 质疑 / 反驳 | `frown` `shake` `hold` | `nod`（被反驳还点头=逻辑矛盾） |
| 提问（问句） | `listen` `brow` | `laugh`（问句配大笑=答非所问） |

> 一句话：**反应是说话方情绪的镜子，不是听者的独角戏。**

## 3. 反应镜头时长经验

基准：**对话/访谈类 cutaway 普遍 1–3 秒，多数落在 1–2 秒。** 再按内容密度/节奏微调：

| 情境 | 时长档 | 依据 |
|------|--------|------|
| 信息密集、语速快、干货段 | 0.6–1.2s | 节奏要紧，反应只"点一下"就回主轨 |
| 常规对话 | 1.0–2.0s | 默认区间 |
| 情绪高点 / 笑点 / 戏剧停顿 | 2.0–3.0s | 让情绪"落地"再走，反应即内容 |
| 遮长台词跳切（T5/T8） | ≥ 接缝两侧各 0.3s，常 1.5–2.5s | 必须完全盖住接缝，留余量防对不齐 |
| 喜剧快切瞬间反应 | 可低至 0.3–0.5s | 刻意快切出"梗"的节奏 |

**两条硬边界：**

- **下限 ≥ 0.5s**：低于约半秒，观众还没看清反应就切走，等于一帧闪烁噪点（喜剧刻意快切除外）。
- **上限 ≤ 3s**（常规对话）：超过 3s，观众会怀疑"说话的人是不是不说了"，且暴露反应片段在循环、动作不自然。

## 4. 密度 / 节奏经验值

| 内容节奏 | 反应镜头密度 | 说明 |
|----------|--------------|------|
| 慢 / 深度访谈 | 每 15–25s 一次 | 主要靠说话片段本身，反应做点缀 |
| 常规双人播客 | 每 8–15s 一次 | 主力区间，约每 2–4 句一个反应 |
| 快节奏 / 综艺感 | 每 5–8s 一次 | 偏密，但仍守"间隔 ≥ 3s"下限 |

**通用经验（均为起点，按内容微调）：**

- **间隔下限 3s**：两个 cutaway 之间，主轨至少留 3s 不被盖，否则观众"喘不过气"、画面焦虑。
- **长台词必覆盖**：任何一方连续说话 **>10s**，中间至少插 1 个听者反应（顺带遮 T8 跳切）。
- **2:1 经验比**：主轨说话画面时长 : 反应覆盖时长 ≈ **2:1 或更高**。反应是调味，不是主菜——超过这个比，主体就糊了。
- **疏密交替，别等距插**：干货密集段可连切两三个反应，随后留一段长主轨"呼吸"。等距插反应像节拍器，反而假。
- **情绪曲线 = 密度曲线**：`开场轻 → 中段稳 → 高潮密`。开场用一次 `two` 建立空间就够，把反应额度留给金句和笑点。

## 5. J-cut / L-cut 与双人镜头的使用时机

| 手法 | 是什么 | 本 skill 怎么用 | 何时用 |
|------|--------|------------------|--------|
| **L-cut（视觉版）** | 声音继续，画面已切到听者 | **本 skill 的主力**——§0 的"盖画面不盖声音"天然就是 L-cut：主轨声音不断，上层盖听者反应 | 几乎所有 cutaway（T1–T8） |
| **J-cut（思路借用）** | 下一位说话者的声音先于画面进入 | 音画一体片段**难真做 J-cut**（声画绑死）。退而求其次：在剪映里把下一说话片段**音频略微前移**，或在当前句末尾盖一个对方 `hold`/`lean` 反应制造"准备抢话"的预期 | 想要"自然抢话""无缝接话"的真实感时，偶尔用，别滥用 |
| **硬切** | 无过渡直接切 | **默认且几乎唯一的转场**——决策表"转场"列恒为硬切 | 全部对话类 cutaway |
| **双人镜头 `two`** | 两人同框（`two_shot.png`） | 当"标点"用：开场、换话题、长段落收尾建立一次空间关系 | T9；别频繁用——双人同框讲话音画一体最不稳，台词靠单人正打承载 |

**铁律：对话类 cutaway 不加溶解/划像/炫技转场。** 加了显廉价、拖节奏，且暴露这是拼接而非真实多机位。

## 6. 雷区清单（AI 自检，逐条核对再出表）

| 雷区 | 判定（怎么发现） | 规避 |
|------|------------------|------|
| **跳切（jump cut）** | 同一人同机位两片段直接相接（Step 3 标记的接缝） | T8 强制：用对方反应盖住接缝，cutaway ≥ 接缝两侧各 0.3s |
| **反应过载** | 平均间隔 < 3s、几乎每句都切 | 执行 §1 去重 + §4 密度：删低优先级，只在金句/笑点/长台词密集覆盖 |
| **口型穿帮** | 覆盖片段里听者在动嘴/说话（明明声音是对方的） | 反应素材必须"无台词、不动嘴"（生成侧约束见 prompt-recipes）；且已静音 |
| **情绪错位** | 严肃台词配 `chuckle`、玩笑配 `frown`、被反驳配 `nod` | §2 语义对齐铁律：token 情绪 = 说话方情绪标签，逐行核对 |
| **越轴（180°）** | 反应片段视线方向与主轨说话方关系不符（A 单人首帧看左，但约定 A 看右） | 三视图首帧锁定朝向（A 看右 / B 看左），反应片段复用同一单人首帧 |
| **机位/光照不一致跳** | 反应片段盖上主轨时画面一闪 | 反应片段与说话片段**同首帧、同机位、同光照、同景别**（Step 2 复用规则） |
| **同素材重复露馅** | 同一条反应片段被反复贴在多个窗口、动作雷同 | §2 每类多备 1–2 条轮换；`listen` 类可循环/倒放铺多个窗，但近邻窗口换不同条 |
| **盖到重读关键帧** | cutaway 压住说话方一个重要表情/口型重音/金句的"重读字" | cutaway 落在**句末/换气/停顿**，不压重读字（见 §3 落点） |
| **音画双声串台** | 覆盖片段忘静音 | 决策表"静音"列恒为「是」（§0 红线），出表前全列扫一遍 |

## 7. 一行流程回顾（把规则编码成 cutaway 候选）

```
逐行扫对话行表
  → 命中 T1–T9 哪条？（看标点/关键词/连续说话时长/拼接缝）
  → 切给谁？（被问方/听者/被反驳方/双人）
  → 选 token？（§2 词表，按说话方情绪标签对齐）
  → 盖多久？（§3 时长档）
  → 落哪？（句末/换气，不压重读字）
  → 去重（§1 优先级 + §4 间隔≥3s）+ 静音恒为是
  ⇒ 一条 cutaway 候选
```

候选列表交回 SKILL.md：去重归并 → Step 5 素材库清单；落点精确到秒 → Step 7 剪辑决策表。

---

> **本 playbook 一句话**：反应镜头不是临场发挥，而是从对话稿文本**可判定地算出来**的——T1–T9 决定何时切、词表决定切什么语义、§3/§4 决定切多久多密、§0/§6 锁死"盖画面不盖声音 + 静音 + 硬切 + 不动嘴"四条不可破的红线。
